驱动版本 595.71.05 (2026-04-28) — Wayland 挂起恢复修复

2026-04-28

更新: open-gpu-kernel-modules 595.71.05 是本周最新稳定版。唯一变更：修复 Wayland 下 OpenGL 应用从挂起恢复后停止渲染的 bug。延续上周 modeset=1 默认启用的 DRM/KMS 生产就绪方向。

Stable Bug Fix
来源: GitHub Release | NVIDIA Driver Page | Release Notes

版本详情

595.71.05 是一个小型补丁版本，唯一的官方变更说明是："Fixed a bug that could cause OpenGL applications to stop rendering after resuming from suspend when running on Wayland." 这延续了 595.71 系列的 DRM/KMS 生产就绪路线。提交由 aritger 于 2026-04-28 完成。适用于 x64 (AMD64/EM64T) 和 ARM64 (aarch64) Linux 平台。RPM Fusion 包也已同步更新 (如 kmod-nvidia-open-595.71.05-1.fc44.x86_64)。

2026-05-05 新 Issue

新 Bug: Vulkan 导出的 dmabuf 无法通过 ibv_reg_dmabuf_mr 导入 mlx5 (ConnectX-7)，scatter-gather 表检索失败。疑似涉及 BAR1 宿主机可见内存区域。仅出现在开源内核模块变体。环境：RTX PRO 5000 Blackwell + 驱动 595.71.05。

New Bug DMA-BUF / Memory
来源: Issue #1130

问题详情

用户 @bmerry 报告 Vulkan 导出的 dmabuf 无法通过 ibv_reg_dmabuf_mr 被 mlx5 (ConnectX-7) 导入。scatter-gather 表检索失败，疑似涉及 BAR1 宿主机可见内存区域。关键信息：此问题仅出现在开源内核模块变体，闭源驱动无此问题。环境：驱动 595.71.05，Ubuntu 24.04.4 LTS，内核 6.17.0-20-generic，RTX PRO 5000 Blackwell。涉及文件路径：kernel-open/nvidia-drm/nvidia-drm-gem-nvkms-memory.c。这揭示了开源模块在跨设备内存共享 (GPU → NIC DMA) 场景下的兼容性缺口。

技术趋势分析

UVM 稳定性危机 — 多 Issue 持续开放

本周 Issue #1130 新开放 (Vulkan dmabuf 导入失败)，加上已持续 3 周+ 的 #1108 (UVM 内核恐慌)、#1125 (GSP 挂起恢复)、#1117 (Linux 7.0 s2idle)，以及论坛新报告的 580.159 _raw_q_flush NULL 解引用。在 Arch Linux 已将开源模块设为默认驱动的背景下，这些稳定性问题的影响面正在急剧扩大。

需要关注

DRM/KMS 持续稳定 — Wayland 修复

595.71.05 专门修复了 Wayland 下 OpenGL 应用挂起恢复后停止渲染的问题，延续了 modeset=1 默认启用的 DRM/KMS 生产就绪方向。这表明 NVIDIA 正在持续投入开源模块的桌面体验优化。

积极进展

Vulkan 开发者 Beta — 实验性特性推进

595.44.06 Vulkan Developer Beta 发布，含 VK_KHR_maintenance11、VK_EXT_descriptor_heap 性能改进、BDA 着色器优化等。同时 Mesa 26.0 NVK 驱动也在推进计算 warp size 改进。开源 Vulkan 生态系统正在多轨并行发展。

积极进展

短期趋势 (2026 Q2)

  • Issue #1130 (dmabuf + mlx5) 预计将被优先处理 (涉及 GPU-NIC DMA 互操作)
  • #1108 (UVM 内核恐慌) 需要紧急修复 (已开放 3 周+)
  • 595.71.05 的 Wayland 挂起恢复修复将提升桌面用户体验
  • Linux 6.19 最终版发布前需要解决驱动兼容性 (580.xx / 590.xx)
  • 595.44.06 Vulkan Beta 的实验特性将在 GA 版本中逐步集成
  • Astera Scorpio X 采样进展，2026 年底量产将影响互连生态

中期趋势 (2026-2028)

  • NVIDIA Nova (Rust DRM 驱动) 逐步推进，可能成为长期上游方案
  • 第六代 NVLink (3.6 TB/s) Rubin 平台开源驱动适配
  • PCIe 6.0 作为 NVLink 替代方案的生态位扩展 (Astera Scorpio X)
  • Linux 发行版 NVIDIA 开源驱动默认采用成为行业标准
  • D3 电源管理在开源模块中的实现 (当前仍缺失)
  • DRM/KMS 完全取代传统 X 驱动路径

本周核心洞察

1. UVM 稳定性问题加速累积 — 开源模块的信任危机

本周 #1130 新开放，加上持续 3 周+ 的 #1108 (内核恐慌)、#1125/#1117 (挂起恢复)、论坛新报告的 580.159 NULL 解引用，以及 #979 (eGPU 硬锁死)。在 Arch Linux 默认采用开源模块的背景下，用户基数扩大意味着这些问题的影响面正在急剧扩大。NVIDIA 需要加速修复节奏，否则可能影响开源驱动的采用信心。

2. DMA-BUF + NIC DMA 互操作 — 开源模块的兼容性缺口

Issue #1130 揭示了开源模块在跨设备内存共享 (GPU Vulkan dmabuf → ConnectX-7 NIC DMA) 场景下的兼容性缺口。该问题仅出现在开源模块变体，闭源驱动无此问题。随着多 GPU + DPU/SmartNIC 部署增加，这类跨设备 DMA 互操作需求将显著增长，是开源模块需要补齐的关键能力。

3. DRM/KMS 桌面体验持续优化 — Wayland 挂起恢复修复

595.71.05 专门修复了 Wayland 下 OpenGL 应用从挂起恢复后停止渲染的 bug，延续了 modeset=1 默认启用的生产就绪方向。595.44.06 Vulkan Beta 也引入了大量实验性特性。这表明 NVIDIA 在开源模块的桌面用户体验和开发者体验上持续投入。

4. Astera Scorpio X — PCIe 6.0 作为 NVLink 替代方案崛起

Astera Labs 的 Scorpio X PCIe 6.0 Fabric 交换机 (320 通道, 5.12 TB/s) 将于 2026 年底量产，并将支持 NVIDIA 的 NVLink Fusion 框架。虽然带宽低于 NVSwitch 6 的 14.4 TB/s，但其供应商中立性为不想完全锁定 NVIDIA 专有互连的数据中心提供了可行替代方案。

5. 多轨并行的开源 Vulkan 生态系统

NVIDIA 官方开源内核模块 (595.44.06 Vulkan Beta)、Mesa NVK 驱动 (26.0 warp size 改进)、Nova Rust DRM 驱动 (Linux 7.1) 三个项目正在并行发展。短期内官方模块是生产首选，但中长期 Nova (Rust + 主线内核) 可能成为上游标准。这种多轨策略增加了生态系统的韧性但也带来了碎片化风险。
