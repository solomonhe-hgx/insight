Feature 580.173.02 内存管理变更

CDMM — Coherent Driver-Based Memory Management (GB200)

新功能

创新点: 580.173.02 引入 CDMM (Coherent Driver-Based Memory Management)，允许驱动直接管理 GPU 内存，避免 OS 级别的内存上线/下线操作。这是 GB200 平台的关键内存管理架构变更。

来源: Release Notes 580.173.02

CDMM 是 NVIDIA 针对 GB200 平台的新型内存管理模式。传统方式中，GPU 内存需要操作系统进行页面上线 (onlining) 操作，而 CDMM 将这一管理权直接交给 GPU 驱动，减少了 OS 干预和潜在的内存热插拔问题。

#1222 — Blackwell RTX 5060 Ti (sm_120): 关机挂起 ~2 分钟

Open

RTX 5060 Ti 在 nv_pci_shutdown → RestoreConsole 路径关机挂起约 2 分 9 秒，I/O 端口 0x600c/0x6008 卡死。sm_120 架构新问题。

来源: GitHub Issue #1222

硬件: RTX 5060 Ti (Blackwell, sm_120)。软锁up发生在 nv_pci_shutdown 期间，I/O 端口访问卡死。这是 Blackwell 消费级 GPU 在开源模块中的新报告问题。

#1221 — 内核 oops: os_get_euid NULL 指针解引用

Open

RTX 4060 Max-Q + 驱动 610.43.02 + Gentoo Linux 7.1.1 + sway 环境下，os_get_euid 地址 0x18 处 NULL 指针解引用导致系统冻结。

来源: GitHub Issue #1221

NULL 指针解引用发生在 os_get_euid 函数中，偏移量 0x18 暗示某个结构体指针为 NULL。该问题在 sway Wayland 合成器下触发，可能与 DRM 模式设置路径相关。

#1208 — RTX 5090 (sm_120): MMU FAULT_PDE (Xid 31) 反复触发

Open

RTX 5090 在约 20 次 CUDA 编译后触发 MMU 页目录项 (PDE) 故障 (Xid 31)。直接与 MMU 页表管理相关。

来源: GitHub Issue #1208

硬件: RTX 5090 (Blackwell, sm_120)。在单个 CUDA context 中约 20 次编译后触发 Xid 31 MMU FAULT_PDE。临时解决方案: 复用 JIT、清除缓存或每 ~15 次运行重启。这表明 MMU 页表在反复 JIT 编译后可能出现泄漏或损坏。

#1212 — S3 恢复时 nvidia_uvm NULL 指针解引用 (内核恐慌)

Open

Dell Precision 5510 (Quadro M1000M Maxwell) 在 S3 恢复时 nvidia_uvm 模块触发内核恐慌。目前唯一解决方案是禁用 NVIDIA GPU。

来源: GitHub Issue #1212

UVM 模块在 S3 恢复路径中的 NULL 指针解引用。Maxwell 架构 GPU + 旧平台。该问题在闭源驱动中可能不存在，暗示开源 UVM 模块的电源管理路径有回归。

#1149 — B200 NVLink 间歇性链路故障 (xid 149)

Discussion

B200 单 SIM 卡出现间歇性 NVLink 故障。NVML:Unable to retrieve Nvlink information as all links are inActive + xid 149。重启可临时修复。

来源: GitHub Discussion #1149

Blackwell B200 数据中心 GPU 的 NVLink 链路间歇性失效。所有 links 变为 inActive 状态。580.173.02 中新增的 ABM Fabric Health Mask Update 机制可能正是针对此类问题的预防性修复。

本周核心洞察

1. ABM Fabric Health Mask Update 是本周最重要的代码变更

新增 135+ 行 Blackwell NVLink 代码实现了 fabric 健康监控的自动化机制：60 秒延迟定时器检查 → clique 降级 → fabric 健康状态覆盖。这直接回应了社区报告的 B200 NVLink 间歇性故障 (Discussion #1149) 和 A100 GSP 超时 (Issue #1118)。GSP RPC 回调的加入表明该功能在 GPU System Processor 层面深度集成。

2. CDMM 标志着 GPU 内存管理架构的重要转变

Coherent Driver-Based Memory Management 将内存管理从 OS 层面转移到驱动层面，这是对传统 Linux 内存管理模型的重大偏离。GB200 平台率先采用，预计将扩展到更多 Grace/Blackwell 融合平台。配合 DMA 方向重构和只读映射支持，NVIDIA 正在系统性地重构开源模块的内存管理栈。

3. Blackwell 消费级 GPU 稳定性是最大风险

本周 21 个活跃 issue 中，绝大多数涉及 Blackwell 消费级 GPU (RTX 5060 Ti/5080/5090)。GSP heartbeat timeout (Xid 109/119)、MMU fault (Xid 31)、GPU lost from bus 等问题高频出现。sm_120 架构的开源驱动成熟度明显落后于数据中心版本。如果这些问题不解决，可能阻碍 Blackwell 消费级 GPU 在 Linux 工作负载中的采用。

4. Linux 7.2 兼容性适配显示开源模块的长期维护承诺

86 行新增兼容层代码处理 drm_atomic_state → drm_atomic_commit 重命名，表明 NVIDIA 在积极跟踪 Linux 内核上游变更。这对 Arch Linux 等滚动发行版用户尤其重要，也是 Arch Linux 将开源模块设为默认的前提条件。
