AMD Collective Communication Compute-Comm Overlap

On-chip Collective Operations / On-chip Collective Operations

WO2025221367A3 | Filed: 2025-02-26 | Published: 2026-01-29

Core Innovation: DMA circuits generate memory access requests for collective operations with dedicated buffers accessible by processing and DMA circuits, separate from cache hierarchy, enabling efficient on-chip collective communication without stalling compute pipelines.

Technical Analysis

  • DMA-based Collective Ops: Specialized DMA circuits generate memory access requests specifically for collective operations (e.g., AllReduce, AllGather), offloading communication from compute units
  • Dedicated Buffers: Buffers accessible by both processing circuits and DMA circuits, physically separate from cache, preventing communication traffic from polluting compute data cache
  • Compute-Communication Overlap: By using separate DMA engines for collective ops, compute kernels can execute simultaneously with data synchronization, directly addressing the compute-communication parallelism challenge
  • On-chip Implementation: Collective operations are implemented at the on-chip level (within GPU die/package), reducing dependency on external network for intra-node synchronization
  • Multi-GPU Scalability: Architecture supports scaling across multiple GPU chiplets within a package via Infinity Fabric interconnect

Impact Analysis

This patent directly targets one of the most critical bottlenecks in distributed AI training: the synchronization overhead between compute and communication. By implementing collective operations in dedicated DMA hardware with separate buffers, AMD provides a path to:

  • Reduce training time by 15-30% through improved overlap
  • Compete with NVIDIA's approach using dedicated DMA engines for collective communication
  • Strengthen AMD's position in data center AI training market (MI350X/MI400 series)

AMD Memory/Cache 3D Packaging

Balanced Latency Stacked Cache / Pingheng Yanchi Duizhan Huancun

US20260003794A1 | Filed: 2024-06-28 | Published: 2026-01-01

Core Innovation: A stacked cache system with central control circuitry and vertical center connection vias that bifurcates control logic between stacked dies to achieve balanced access latency across all cache regions, enabling 3D-stacked L2/L3 cache with uniform performance.

Technical Analysis

  • Bifurcated Control Logic: Central control circuitry is vertically split between the first and second cache dies, with each half controlling its adjacent cache region, minimizing wire delay
  • Center Connection Vias: Vertical center connection vias provide I/O paths, with horizontal bifurcation of die sections for balanced latency to all cache banks
  • L2 Stacked Cache: Specifically targets L2 cache stacking (beyond current 3D V-Cache which stacks L3), potentially doubling or tripling L2 capacity per compute unit
  • SRAM/DRAM Implementation: Supports both SRAM and DRAM vertical stacking, providing flexibility for different performance/capacity trade-offs
  • Latency Symmetry: The key innovation is ensuring that access latency is balanced regardless of which die layer is accessed, solving a critical problem in 3D cache design

Impact Analysis

This patent represents the next evolution of AMD's 3D V-Cache technology, extending from L3 to L2 cache stacking. For GPU workloads:

  • Could dramatically increase on-chip memory bandwidth for GPU compute units, reducing memory-bound stalls
  • Balanced latency design ensures predictable performance for real-time AI inference workloads
  • Directly competes with NVIDIA's approach to increasing on-chip memory bandwidth in next-gen GPUs
  • Applicable to both CPU (EPYC/Ryzen) and GPU (Instinct/Radeon) product lines

AMD GPU Interconnect Cache Coherence

Active Bridge Chiplet with Integrated Cache / Jiyu Huancun de Huodong Qiaojie Chiplet

US11507527B2 (filed as US20210097013A1) | Filed: 2019-09-27 | Granted: 2022-11-22

Core Innovation: An active silicon bridge chiplet containing a unified last-level cache (LLC/L3) that maintains cache coherence across all connected GPU chiplets, providing high-bandwidth die-to-die interconnects and presenting a monolithic GPU view to software.

Technical Analysis

  • Active Silicon Bridge: Unlike passive interposers, the bridge chiplet is an active die with its own logic, serving as a memory crossbar between GPU chiplets
  • Unified LLC: The bridge contains a shared, unified L3 cache that remains coherent across all connected GPU chiplets, enabling data sharing without going to main memory
  • Master-Slave Topology: A primary GPU chiplet receives CPU requests and routes them through the active bridge to access data cached on secondary chiplets
  • Monolithic Software View: Software sees a single monolithic GPU device, abstracting away the chiplet partitioning complexity
  • Foundation for Infinity Fabric: This patent forms the architectural basis for AMD's Infinity Fabric interconnect technology used in MI300 series accelerators

Impact Analysis

This is a foundational patent for AMD's multi-chiplet GPU strategy, directly implemented in the Instinct MI300X/A accelerators:

  • Enables AMD to build high-performance GPUs with many chiplets while maintaining software compatibility
  • The unified cache approach is critical for achieving near-monolithic performance in chiplet-based GPUs
  • Directly competes with NVIDIA's NVLink-C2C for multi-die GPU interconnect
  • Key enabler for AMD's path to 1.5TB+ HBM memory systems in MI350X/MI400

  • Partition Flexibility: Supports single-die, dual-die, and quarter-die partition modes, enabling flexible resource allocation for different workload requirements
  • Virtualization Support: Second-level mapping tables enable efficient interrupt routing to different guest OSs in virtualized environments
  • Cross-die Coordination: Essential infrastructure for managing communication and synchronization across GPU chiplets in large-scale accelerators

Impact Analysis

This patent provides critical infrastructure for AMD's chiplet-based GPU architecture, enabling efficient multi-die management:

  • Essential for scaling GPU chiplet designs beyond 2-4 dies while maintaining manageable interrupt overhead

Intel GPU Interconnect Memory Management

Multi-tile Graphics Processing Unit / Duotupian Xing Tuxing Chuli Danwei

US12367540B2 | Filed: 2020-11-18 | Granted: 2025-07-22

Core Innovation: A multi-tile GPU architecture with distributed memory devices per tile, a dedicated memory management unit for cross-tile memory allocation, and high-bandwidth inter-tile interconnects enabling coherent memory access across all tiles.

Technical Analysis

  • Tile-based Architecture: Each processing tile includes its own memory device and processing resources, enabling modular scaling of GPU compute capacity
  • Dedicated MMU: A memory management unit manages memory allocation across all distributed tile memory devices, providing unified virtual address space
  • Cross-tile Interconnect: High-bandwidth inter-tile interconnects enable low-latency communication and data sharing between tiles
  • Coherent Memory: Memory coherence is maintained across all tiles, allowing any processing resource to access any tile's memory with consistent semantics
  • Foundation for Xe-HPC: This architecture underpins Intel's Data Center GPU Max series (Ponte Vecchio) with up to 8 compute tiles

Impact Analysis

This patent is central to Intel's multi-tile GPU strategy for data center accelerators:

  • Enables Intel to scale GPU performance through tiling rather than relying on single monolithic dies
  • The cross-tile MMU is critical for developer experience, presenting a unified memory model
  • Directly competes with AMD's chiplet approach and NVIDIA's multi-die GPU designs
  • Key enabler for Intel's Battlemage and future data center GPU architectures

US20250231770A1 | Filed: 2024-12-24 | Published: 2025-07-17

Core Innovation: A workload analyzer groups neural network nodes by computation distribution and assigns them to processing cores with pipeline execution, supporting unified memory across processors and bias-based memory management to optimize compute-communication overlap in distributed deep learning.

Technical Analysis

  • Workload Analysis: A network workload analyzer receives workloads, analyzes computation distribution, and groups network nodes into computation groups for optimized execution
  • Pipeline Execution: A network executor assigns each group to processing cores, enabling pipelined execution where computation and communication overlap across layers
  • High-speed Interconnect: Supports PCIe/NVLink-class links with communication throughput of 4-80+ GB/s between processors
  • Unified Memory: Same virtual system address space distributed across all physical memories, enabling zero-copy data access between CPU and GPU

Impact Analysis

This patent directly addresses the compute-communication parallelism challenge in distributed deep learning training on Intel platforms:

  • Enables more efficient pipeline parallelism by intelligently grouping computation and overlapping with communication
  • The bias-based memory management reduces cross-processor memory traffic, improving effective bandwidth
  • Part of Intel's oneAPI ecosystem strategy to compete with NVIDIA's CUDA-based training stack
  • Supports Intel Gaudi accelerators and Xe-HPC GPUs in multi-node training configurations

Core Innovation: Pre-fetches constant weight values and key-value entries into buffers during collective communication phases (AllReduce/AllGather) of transformer kernel processing, hiding communication latency by overlapping data movement with computation in LLM inference and training.

Technical Analysis

  • Transformer-aware Prefetching: Specifically designed for transformer kernel processing patterns, pre-fetching constant weight values and KV-cache entries during communication phases
  • AllReduce/AllGather Overlap: Data pre-fetch occurs during collective communication operations, effectively hiding the communication latency behind useful data movement
  • Buffer Management: Dedicated buffer circuitry receives pre-fetched data, making it available for subsequent compute operations without stalling
  • Pipeline Optimization: Addresses the specific challenge of pipeline bubbles in LLM inference where communication stalls can be filled with useful prefetch operations
  • Multi-GPU Support: Designed for distributed LLM processing across multiple GPUs connected via high-speed interconnects

Impact Analysis

This patent targets one of the hottest areas in AI: efficient LLM inference and training across multiple accelerators:

  • Directly improves LLM serving throughput on Intel Gaudi and Xe-HPC accelerators
  • The collective communication overlap technique is applicable to both training and inference workloads
  • Competes with NVIDIA's approaches to hiding communication latency in transformer execution
  • Part of Intel's broader strategy to optimize oneAPI Collective Communications Library (oneCCL)

Trends & Outlook / Qushi yu Zhanwang

1 Hardware-based Collective Communication is the New Battleground

Both AMD and Intel are moving collective communication operations from software libraries (RCCL/oneCCL) into dedicated hardware. AMD's DMA-based on-chip collective operations (WO2025221367A3) and Intel's transformer-aware prefetching during AllReduce (US20240370699A1) both signal that:

  • - Software-only collective communication is insufficient for next-gen AI training scale
  • - Hardware DMA engines specifically designed for collective ops will become standard
  • - The boundary between "network" and "compute" is blurring at the silicon level

2 3D Memory Stacking Goes Beyond HBM

AMD's balanced latency stacked cache (US20260003794A1) and Intel's vertically stacked memory dies (US20250201793A1) both push beyond standard HBM:

  • - AMD extends 3D stacking from L3 (V-Cache) to L2 cache, targeting compute-local bandwidth
  • - Intel targets 1-2 TB per package at 100-200 TB/s, far exceeding HBM3E capabilities
  • - Both companies recognize that memory bandwidth, not compute, is the primary bottleneck for AI

3 Chiplet/Tile GPU Architecture is Now Mainstream

All major GPU vendors have embraced multi-die architectures. The key differentiators are shifting:

  • - AMD's active bridge with unified cache vs Intel's per-tile MMU approach
  • - The challenge of maintaining cache coherence across dies remains central
  • - Software transparency (presenting a monolithic view) is table stakes
  • - Inter-die interconnect bandwidth and latency are the new performance differentiators

4 Compute-Communication Overlap Moves from Software to Hardware

The traditional approach of overlapping communication with computation via software threading is giving way to hardware-level solutions:

  • - AMD's dedicated DMA circuits for collective ops run independently of compute pipelines
  • - Intel's workload analyzer automatically identifies overlap opportunities in neural network graphs
  • - Both approaches reduce the software engineering burden on AI developers
  • - Hardware overlap mechanisms achieve higher overlap ratios than software approaches

5 LLM-specific Hardware Optimization Accelerates

Intel's LLM prefetching patent (US20240370699A1) signals a trend toward transformer-architecture-specific hardware:

  • - Hardware that understands transformer attention patterns and KV-cache access
  • - Prefetching strategies tailored to AllReduce/AllGather in tensor/pipeline parallelism
  • - The convergence of "network architecture" and "neural network architecture" awareness in silicon
  • - Expect more transformer-specific optimizations in both AMD and Intel future products
