摘要

本发明公开了一种 Fabric Attached Memory (FAM) 技术，通过将专用内存模块直接连接到高速互连网络 (如 NVLink/NVSwitch)，实现可扩展的 GPU 内存容量，且独立于计算资源。系统提供 TB 级存储和加速吞吐量，无需购买额外处理单元。该架构依赖自动化流量分发和地址转换技术，确保跨分布式内存池的高效硬件利用。FAM 提供了一种将内存从计算资源解聚 (disaggregate) 的方法，使内存容量能够独立于计算容量增长。

来源: Google Patents - US11822491B2

技术背景

在大规模 AI 训练和 HPC 场景中，GPU 显存容量成为主要瓶颈。传统架构中，内存容量与 GPU 计算资源绑定——要获得更多内存，必须购买更多 GPU。这导致：

  • • 内存受限的工作负载被迫购买过剩的计算能力，造成资源浪费
  • • 大规模模型 (如万亿参数 LLM) 需要跨多 GPU 分区模型，增加通信开销
  • • 内存池化需求增长，但现有方案 (如 CXL) 在带宽和延迟上无法满足 GPU 需求
  • • 需要一种高带宽、低延迟的内存解聚方案，充分利用 GPU 已有的互连 fabric (NVLink/NVSwitch)

核心洞察: Fabric attached memory 利用已有的高速互连 fabric (NVLink/NVSwitch) 的优势，将内存模块作为 fabric 上的独立节点连接，而非绑定到特定 GPU。这 leverages fabric 的可扩展性和带宽，同时避免了额外的互连基础设施。

发明概述

本发明提出了一种完整的 Fabric Attached Memory 系统，包含以下核心技术：

1. 地址 Swizzling (随机化)

虚拟地址在传输前转换为物理 fabric 地址。转换过程对特定位范围应用数学熵 (entropy)，有效"随机化链接选择"。这防止流量集中在特定路径上，确保 "no one link is underutilized and no one link is overloaded"，最大化聚合带宽。

2. 流量 Spraying (分发)

源节点在所有可用连接上交错传出请求。这种分发技术确保 "spraying has the effect of load-balancing memory traffic across the different links so that none are overwhelmed"。通过随机化链路选择，流量均匀分布在所有物理链路上，消除热点。

3. 地址 Compaction (压缩)

网络交换机反转初始随机化并压缩传入地址。通过消除交错的地址间隙，匹配单个设备较小的容量。"compaction operates on an unswizzled address"，确保目标 FAM 模块看到 "无空洞的线性地址空间"，实现完整的存储利用率。

4. FAM 控制器架构

控制器使用 "floor swept" (功能裁剪) 的处理单元。非必要的计算路径被永久禁用以节省功耗，而互连和内存接口保持完整。这些控制器作为从设备，"直接处理 NVLink 命令包括原子操作 (atomics)"，无需模拟开销，保持完整的协议兼容性。

详细发明内容

一、FAM 系统架构

整体拓扑

FAM 模块通过高速互连 fabric (NVLink/NVSwitch) 直接连接，作为 fabric 上的独立内存节点。GPU/处理器通过 fabric 访问 FAM，如同访问本地内存一样，使用原生内存访问命令 (load/store/atomic)。

FAM 模块组成

每个 FAM 模块包含：(a) 高容量内存 (DRAM/HBM)；(b) FAM 控制器 (floor swept 处理单元 + NVLink 接口)；(c) 地址转换硬件 (swizzle/compaction 引擎)。FAM 控制器作为从设备，响应 fabric 上的内存访问请求。

交换机路由

网络交换机使用可编程路由条目和 "map slot" 表来引导流量、分配目标标识符和执行分区边界。物理分组可细分为逻辑段，支持虚拟化环境和租户隔离。

二、地址 Swizzling (随机化) 机制

原理

当 GPU 向 FAM 发送内存访问请求时，虚拟地址首先转换为物理 fabric 地址。这个转换不是简单的线性映射，而是对地址的特定位范围应用数学变换 (熵函数)，产生"随机化"的物理地址。这种随机化确保连续虚拟地址被映射到不同的物理链路和 FAM 模块上。

目的

如果没有 swizzling，连续地址会集中到同一条链路和同一个 FAM 模块，导致该链路过载而其他链路闲置。Swizzling 通过随机化确保 "randomizing link selection for a given address"，实现链路级别的负载均衡。这对于顺序访问模式 (如 stream 读写) 和 stride 访问模式特别重要。

技术细节

Swizzling 函数作用于地址的特定比特位，通过 XOR、置换或哈希函数改变地址到链路/模块的映射。变换是可逆的，确保 FAM 控制器可以通过逆操作 (unswizzle + compact) 恢复正确的本地地址。Swizzling 的粒度可以配置，适应不同的 FAM 模块数量和链路拓扑。

// 地址 Swizzling 流程 (简化示意)
virtual_address = [VA_high | VA_mid | VA_offset]
// Step 1: 应用熵函数到特定位
swizzled_bits = entropy_function(VA_mid)
// Step 2: 使用 swizzled bits 选择链路和模块
link_select = swizzled_bits[link_bits]
module_select = swizzled_bits[module_bits]
// Step 3: 构建 fabric 地址
fabric_address = [module_select | link_select | VA_offset]
// 效果: 连续虚拟地址分散到不同链路/模块

三、流量 Spraying (分发) 机制

原理

源 GPU/处理器在发送内存请求时，不固定使用某一条链路，而是将请求交错 (interleave) 到所有可用的出站链路。这与地址 swizzling 配合：swizzling 决定目标 FAM 模块，spraying 决定从源到目标的传输路径。

效果

"Spraying has the effect of load-balancing memory traffic across the different links so that none are overwhelmed." 即使多个 GPU 同时访问同一个 FAM 模块，spraying 也能确保流量通过不同路径到达，避免单条链路过载。这最大化了聚合带宽利用率，确保 "no one link is underutilized and no one link is overloaded"。

实现方式

源节点的内存控制器维护多个出站链路的状态 (队列深度、可用带宽)。发送请求时，根据当前链路状态选择最优路径。支持 round-robin、随机选择或基于负载的动态选择策略。交换机在转发时也可执行 secondary spraying，进一步分散流量。

四、地址 Compaction (压缩) 机制

问题

全局 FAM 地址空间是跨多个物理模块交错的 (由于 swizzling)。但每个单独的 FAM 模块只有有限的本地容量 (如 32GB/64GB)。如果直接将全局地址发送到 FAM 模块，地址范围会远超模块容量，产生大量"空洞" (未映射的地址间隙)。

解决方案

网络交换机 (或 FAM 控制器入口) 在将请求转发到目标 FAM 模块之前，执行地址压缩。首先撤销 swizzling (unswizzle)，恢复原始地址的连续结构。然后压缩 (compact) 地址，消除交错产生的间隙，将全局地址映射到模块的本地线性地址空间。

关键特性

"Compaction operates on an unswizzled address"——压缩只在 unswizzle 之后执行，确保地址的连续性。压缩后，"the target FAM module sees linear address space without holes"，模块可以使用简单的线性地址解码器，无需复杂的地址映射逻辑。这简化了 FAM 控制器设计，降低功耗和延迟。

// 地址 Compaction 流程 (简化示意)
// 输入: 来自 fabric 的全局交错地址
global_fabric_address = [module_id | interleaved_offset]
// Step 1: Unswizzle - 撤销随机化
unswizzled = inverse_entropy(global_fabric_address)
// Step 2: Compact - 消除间隙
local_offset = compact(unswizzled, module_id)
// Step 3: 输出本地线性地址
local_address = [local_offset]
// 效果: FAM 模块看到无空洞的线性地址空间

五、FAM 控制器详细设计

Floor Swept 处理单元

FAM 控制器使用 intentionally restricted ("floor swept") 的处理单元。"Floor sweeping" 是芯片制造中的一种技术：在芯片测试后，禁用部分有缺陷或不需要的功能单元。对于 FAM 控制器，完整的 GPU 计算单元 (如 CUDA Core、Tensor Core) 被永久禁用，只保留互连接口和内存控制器。这大幅降低了 FAM 控制器的功耗和成本，同时保持了完整的 NVLink 协议处理能力。

NVLink 原生支持

FAM 控制器作为从设备，直接处理 NVLink 命令，包括原子操作 (atomics: atomicAdd, atomicCAS, atomicExchange 等)。这不需要软件模拟或协议转换——FAM 控制器在硬件层面原生支持 NVLink 协议。因此 GPU 可以像访问本地显存一样访问 FAM，使用相同的 load/store/atomic 指令，零软件开销。

从设备角色

FAM 控制器不主动发起请求，只响应来自 GPU/处理器的内存访问。它接收 fabric 地址，执行 unswizzle + compact 转换，访问本地内存，返回数据。这种简化的从设备模型减少了控制器复杂度，确保低延迟响应。

六、内存池管理

软件定义分组

存储被组织为软件定义的分组 (groupings)，跨多个 FAM 模块实现并行数据访问。软件 (驱动/运行时) 决定如何将内存分配映射到物理模块。分组可以动态创建、销毁和调整大小，适应不同的工作负载需求。

交换机路由表

网络交换机使用可编程路由条目和 "map slot" 表来引导流量。每个 map slot 条目定义了全局地址范围到物理 FAM 模块的映射。交换机根据 map slot 表将请求路由到正确的目标。这支持动态重配置——当 FAM 模块添加或移除时，只需更新 map slot 表。

虚拟化与租户隔离

物理 FAM 分组可细分为逻辑段 (logical segments)，支持虚拟化环境和租户隔离。每个虚拟机或租户获得独立的逻辑 FAM 空间，无法访问其他租户的数据。这通过交换机的分区边界 enforcement 和 FAM 控制器的访问权限检查实现。

条带化 (Striping)

软件将扩展的内存池分区为并行段 (parallel segments/stripes)，实现跨多个物理模块的同时访问。条带大小可以配置——小条带适合随机访问 (更好的负载均衡)，大条带适合顺序访问 (减少跨模块访问)。这与地址 swizzling 配合，确保条带均匀分布。

技术影响与应用场景

AI 大模型训练

万亿参数 LLM 需要 TB 级内存。FAM 允许将内存扩展到 GPU 本地显存之外，无需额外的模型分区。结合 NVLink 高带宽，FAM 访问延迟远低于基于以太网的内存池化方案。

HPC 科学计算

大规模科学模拟 (气候模拟、流体力学) 需要处理超大数据集。FAM 提供高带宽、大容量的内存池，支持单 GPU 逻辑地址空间访问 TB 级数据。

多租户云 GPU

云服务商可以动态分配 FAM 容量给不同的虚拟机租户。逻辑段隔离确保租户间数据安全。FAM 控制器使用 floor swept 芯片降低硬件成本。

内存解聚趋势

FAM 代表了内存解聚的一种高性能路径，与 CXL 内存池化形成互补/竞争。FAM 利用现有 GPU fabric (NVLink)，而 CXL 需要额外的互连基础设施。

核心洞察

US11822491B2 是 NVIDIA 在内存解聚领域的核心专利。它巧妙利用了 NVIDIA 已有的 NVLink/NVSwitch 互连优势，将内存模块作为 fabric 上的独立节点，而非绑定到特定 GPU。通过 Swizzling (地址随机化)、Spraying (流量分发) 和 Compaction (地址压缩) 三项核心技术，实现了 TB 级内存池的高效管理和访问。FAM 控制器使用 floor swept 芯片降低成本的思路也极具商业智慧——利用 GPU 芯片中的"边角料"构建专用内存控制器。该专利与 Intel CXL 4.0 形成竞争/互补关系，代表了两种不同的内存池化路径。
