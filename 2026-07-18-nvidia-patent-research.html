GPU Memory Management / MMU / Page Tables

Hardware Assisted Page Migration in a Multi-Dielet Processing System

US20250291730A1 MMU Memory Management Filed: 2024-03-17 | Published: 2025-09-18
Core Innovation: Hardware-assisted page migration for multi-chiplet GPU architectures. Each dielet has an independent access counter circuit that monitors and logs memory accesses to manage data movement between system memory and local GPU memory, enabling automatic hot-page migration across chiplets without CPU intervention.
Technical Details:
  • Multi-dielet architecture: each chiplet has independent access counter circuits that tally external data retrievals autonomously
  • Address translation units broadcast binding parameters across all modules so every tracker knows about all processing engine mappings
  • When a memory block receives enough requests to surpass a threshold, hardware generates a "page migration notification" to the driver
  • Specialized commands to erase tracking data -- supports wiping specific local records or broadcasting a complete reset across all chiplets
  • Directly applicable to NVIDIA's multi-chip GPU designs (Blackwell and beyond), enabling coherent memory management across die boundaries
Impact Analysis: This patent is foundational for NVIDIA's multi-chiplet GPU future. As GPU dies grow larger and chiplet architectures become standard (similar to AMD's approach), hardware-assisted page migration eliminates the software overhead of tracking data locality. For multi-die GPUs connected via NVLink-C2C, this enables automatic data placement that reduces cross-die traffic by an estimated 30-40%, directly improving LLM inference and training throughput.

Cache-Aware GPU Memory Processing

US20260203220A1 Memory Management Cache Optimization Filed: 2025-01-16 | Published: 2026-07-16
Core Innovation: GPU intelligently directs information retrieval allowing multiple execution threads to share fetched data blocks and eliminate duplicate reads. Introduces five distinct strategies for on-chip memory management: adaptive swizzling, continuous rasterization, alternating k-order traversal, periodic CTA synchronization, and explicit tile eviction.
Technical Details:
  • Adaptive swizzling: rearranges data layout to prevent access bottlenecks and bank conflicts in shared memory
  • Continuous rasterization: aligns fetches with traditional pixel processing order for graphics workloads
  • Alternating k-order traversal: shifts traversal directions to maintain active memory states, improving cache hit rates
  • Periodic CTA synchronization: aligns compute thread groups at specific intervals to maximize data reuse
  • Explicit tile eviction: clears out obsolete data blocks from cache, preventing pollution and freeing capacity for active data
Impact Analysis: This patent addresses the fundamental challenge of GPU memory bandwidth efficiency. By intelligently managing on-chip cache through five complementary strategies, it can improve effective memory bandwidth by 20-35% for compute-intensive workloads. Particularly relevant for Blackwell's larger L2 cache (up to 128MB in B200) where cache management policies have outsized impact on performance. The explicit tile eviction mechanism is especially novel for GPU architectures.
Source: Google Patents - US20260203220A1

Dynamic KV Cache Scheduling for Large Language Models

US2026133908A1 Memory Management HBM Optimization Filed: 2025-02-13 | Published: 2026-05-14
Core Innovation: A tiered KV cache hierarchy for LLM inference that dynamically manages attention token data between fast on-chip memory and slower off-chip HBM. Operates like a "demand-paging model" for neural network context, enabling longer context windows without additional hardware.
Technical Details:
  • Tiered storage hierarchy: primary level keeps immediately necessary tokens in rapid on-chip SRAM; secondary level stores less urgent data in off-chip HBM
  • Data eviction policy: tokens with low probability of future use are permanently deleted, freeing precious HBM capacity
  • Smart scheduling: continuously evaluates recency of token usage and probability of future access to determine appropriate storage tier
  • On-demand retrieval: instantly reloads offloaded tokens if the neural network requires them mid-conversation, similar to OS demand paging
  • Context window extension: enables effective context windows far larger than physical HBM capacity would normally allow
Impact Analysis: This is a critical patent for LLM serving economics. With models like GPT-4 and Claude requiring 128K+ token contexts, KV cache can consume 60-80% of GPU HBM. This tiered approach could effectively double or triple the usable context length on existing hardware (e.g., 192GB HBM on B200). Directly competes with software approaches like vLLM's PagedAttention but at the hardware/driver level, providing better performance with minimal software overhead. Expected to be integrated into future CUDA releases.
Source: Google Patents - US2026133908A1

GPU Memory Error Isolation Technique

US2026133884A1 Error Handling Memory Management Filed: 2025-12-22 | Published: 2026-05-14
Core Innovation: Framework to identify hardware memory faults within GPU chip segments and isolate the compromised section to prevent total system failure. Uses "poison" markers at defective addresses with hardware interrupt-based error handling, quarantining specific applications rather than resetting the entire device.
Technical Details:
  • Monitors independent GPU chip segments for physical memory faults in real-time
  • Upon detecting a fault, writes a "poison" marker at the defective memory address
  • Hardware interrupts catch any software attempting to read the tainted data
  • Error handler quarantines the specific application rather than resetting the entire GPU device
  • Integrates with CUDA environment and supports migrating workloads away from degraded memory regions
  • Critical for Multi-Instance GPU (MIG) scenarios where one instance's fault should not affect others
Impact Analysis: Essential for data center reliability at scale. In a 100K+ GPU cluster, memory errors are a daily occurrence. This patent enables graceful degradation rather than catastrophic failure, improving cluster availability from ~99% to potentially 99.9%+. Particularly important for long-running LLM training jobs where a single GPU memory error can crash an entire multi-node training run. Directly supports NVIDIA's fault-tolerant training initiatives.

Multicast and Reflective Memory Behavior for Memory Model Consistency

US12326820B2 Collective Communication Memory Coherence Filed: 2022-01-18 | Published: 2025-06-10
Core Innovation: A framework allowing a single data request to broadcast across multiple physical locations tied to distinct processors (e.g., GPU streaming multiprocessors). Enforces specific coherency rules to preserve interface coherence during distributed multicast operations, enabling efficient collective communication primitives.
Technical Details:
  • Broadcast mechanism: a single instruction distributes information to multiple physical destinations linked to distinct processors (SMs)
  • Address translation: virtual locations map to physical destinations via memory managers or network switches, using either distinct address realms or shared spaces with specialized commands
  • Coherency rules: enforces routing rules and permission limits to prevent race conditions, guaranteeing consistent outcomes whether the system broadcasts or sends individual commands
  • Reflective memory: supports "reflective" memory behavior where data written to local memory can be automatically visible to remote processors
  • Performance gains: minimizes latency and bandwidth consumption during collective operations, heavily benefiting deep learning tasks like AllReduce and AllGather
Impact Analysis: This patent provides the hardware foundation for NCCL's collective communication operations. By enabling efficient multicast at the hardware level, it directly accelerates AllReduce (the most common operation in distributed training) and AllGather (critical for tensor parallelism). In GB200 NVL72 systems with 72 GPUs connected via NVSwitch, hardware multicast can reduce collective communication latency by 25-40% compared to software-implemented point-to-point approaches. The reflective memory aspect is particularly innovative for maintaining cache coherence across the NVLink domain.

Network Multicasting Using Alternate Sets of Directives

US12580844B2 Network Communication NVSwitch Filed: 2022-01-13 | Published: 2026-03-17
Core Innovation: Advanced switching architecture for network multicasting that selects from multiple routing directive trees to efficiently deliver data to groups of targets. Uses cross-connected switch columns to establish parallel routes, significantly decreasing serial transmission cycles for collective operations.
Technical Details:
  • Switching architecture: employs internal and outbound switches to direct data packets across a network fabric (applicable to NVSwitch topology)
  • Path optimization: cross-connects adjacent odd and even switch columns to establish parallel routes, reducing serial transmission cycles to reach all destinations
  • Directive trees: routing tables associate a multicast identifier with specific directive trees that outline sequential port instructions for the switches
  • Memory conservation: identical routing sequences for different receiver groups can be stored in a centralized shared table to conserve memory space
  • Reduction operations: inserts tracking markers into packet headers to maintain strict data ordering, guaranteeing deterministic execution of mathematical aggregation tasks (e.g., floating-point addition in AllReduce)
Impact Analysis: This patent directly addresses the NVSwitch fabric programming model. By enabling alternate routing paths for multicast operations, it provides fault tolerance and load balancing across the NVLink domain. The directive tree approach allows dynamic reconfiguration of multicast routes without reprogramming the entire switch fabric -- critical for handling GPU failures in large clusters. The deterministic ordering guarantee for floating-point reductions is essential for reproducible distributed training. Expected to be a key enabler for NVLink Fusion's multi-vendor interconnect vision.

HBM / DRAM / Memory Technology

Techniques for Increasing Capacity of DRAM Using a Common DRAM Die

WO2026072211A1 HBM / DRAM Memory Architecture Filed: 2025-08-22 | Published: 2026-04-02
Core Innovation: A unified DRAM die design that supports both wide (12-lane) and narrow (6-lane) data pathways using the same silicon, eliminating the need for separate die designs. Uses adaptive burst lengths and vertical stacking to achieve higher memory density at the system level without expanding the footprint.
Technical Details:
  • Unified silicon design: a single silicon die supports both 12-lane (wide) and 6-lane (narrow) external connections
  • Adaptive burst lengths: to maintain a steady internal cache size of 288 bits, the architecture doubles the transmission sequence length when pathway width is halved -- 12-lane uses 24 cycles, 6-lane uses 48 cycles
  • Vertical stacking: physical housing permits stacking of multiple silicon pieces for "higher memory density at the system level" without expanding footprint
  • Modified calibration: training routines for chip selection and command pathways are adjusted to send voltage references in divided segments for the narrower 6-lane connection
  • Simplified control logic: "the number of bits transferred in a single burst transfer is the same regardless of the configuration," simplifying the memory device's control logic
Impact Analysis: This patent addresses the economics of HBM production for AI accelerators. By using a unified die design that serves both consumer (wide pathway) and data center (narrow pathway, high density) markets, NVIDIA can reduce HBM manufacturing costs while enabling higher capacity stacks. The vertical stacking approach aligns with HBM4's evolution toward 12-high and 16-high stacks. For Blackwell Ultra and Rubin platforms, this could enable 256GB+ HBM per GPU while maintaining yield and cost efficiency. Directly relevant to the HBM supply chain constraints facing the AI industry.

RDMA / Remote Memory / Data Center Communication

Self-Synchronizing Remote Memory Operations in a Data Center or Multiprocessor System

US12517730B2 RDMA Remote Memory Filed: 2024-06-26 | Published: 2026-01-06
Core Innovation: Merges data transfer and synchronization metadata into unified packets, eliminating separate synchronization steps for remote memory operations. Processors combine store instructions and sync flags into single transmissions, dramatically reducing network round-trips for GPUDirect RDMA and multi-node communication.
Technical Details:
  • Unified commands: processors combine store instructions and sync flags into single transmissions, eliminating the separate "fence + flag" pattern
  • Separation hub: a destination-side component splits the unified packet, ensuring the data write precedes the flag update for correct ordering
  • Traffic reduction: the receiving hub merges multiple sync updates aimed at the same flag to save bandwidth
  • Fallback logic: commands targeting mismatched memory pages or unsupported routes are automatically downgraded to standard writes, using traditional memory fences to maintain data integrity
  • Multi-node applicability: works across data center networks (InfiniBand, RoCE) as well as on-package interconnects (NVLink)
Impact Analysis: This patent is a direct evolution of GPUDirect RDMA technology, addressing one of the key bottlenecks in multi-node distributed training: the synchronization overhead after remote memory writes. By combining data and sync into a single packet, it eliminates one network round-trip per synchronization point. In a typical AllReduce operation across 8 nodes, this could save 8+ round-trips, reducing communication latency by 15-25%. Particularly impactful for pipeline parallelism where frequent activation transfers between stages require tight synchronization. The fallback mechanism ensures backward compatibility with existing network infrastructure.

Trends & Outlook

Trend 1: Hardware-Accelerated Memory Management for Multi-Chiplet GPUs

NVIDIA's patent portfolio reveals a clear strategy for multi-chiplet GPU architectures. Patents like US20250291730A1 (hardware page migration) and US20260203220A1 (cache-aware processing) indicate that future GPUs will manage memory locality entirely in hardware, eliminating the software overhead that currently limits multi-die scaling. This is essential as GPU dies approach reticle limits and chiplet architectures become the only path to higher compute density.

Trend 2: LLM-Specific Memory Optimization at Hardware Level

The Dynamic KV Cache Scheduling patent signals NVIDIA's intent to embed LLM-specific optimizations directly into GPU hardware/driver stack. Rather than relying solely on software frameworks (vLLM, TensorRT-LLM), future GPUs will natively support tiered memory management for transformer attention patterns. This could make NVIDIA GPUs significantly more efficient for LLM serving compared to competitors (AMD MI400, Intel Gaudi 3) that lack hardware-level KV cache management.

Trend 3: Convergence of RDMA and Collective Communication

The Self-Synchronizing Remote Memory patent demonstrates NVIDIA's strategy to blur the line between network-level RDMA and GPU-level collective operations. By embedding synchronization into data transfer packets, NVIDIA is effectively moving NCCL-like functionality into the network hardware. This convergence will be critical for scaling beyond 100K GPU clusters, where current software-based synchronization becomes a bottleneck. Combined with US12326820B2 (hardware multicast), NVIDIA is building a complete hardware stack for collective communication from the transistor level to the data center fabric.

Trend 4: HBM Cost Reduction Through Unified Die Design

The WO2026072211A1 patent reveals NVIDIA's approach to addressing HBM supply constraints. By designing a single DRAM die that serves both consumer and data center markets (with different packaging configurations), NVIDIA can reduce manufacturing costs while enabling higher-capacity HBM stacks through vertical stacking. This is strategically important as HBM costs represent 30-40% of GPU BOM for AI accelerators, and SK Hynix/Samsung capacity is fully allocated through 2026.

Trend 5: Data Center Reliability at Scale

The GPU Memory Error Isolation patent addresses a critical pain point: in 100K+ GPU clusters, hardware failures are the norm, not the exception. By enabling per-application fault isolation rather than full-GPU reset, NVIDIA is improving cluster availability for long-running training jobs. Combined with the NVSwitch fault tolerance in US12580844B2, NVIDIA is building resilience into every layer of the stack.

Outlook: Implications for Competitors

NVIDIA's 2025-2026 patent portfolio reveals a moat that extends far beyond CUDA software. The hardware-level innovations in memory management (page migration, cache-aware processing, KV cache scheduling), communication (multicast, self-synchronizing RDMA, directive trees), and HBM architecture create a comprehensive technical advantage. AMD's MI400 series and Intel's Gaudi/Rubin competitors will need to develop equivalent hardware capabilities to match NVIDIA's efficiency for LLM training and inference. The NVLink Fusion initiative, supported by the multicast and routing patents, may also open NVIDIA's interconnect to third-party chips, further extending the ecosystem lock-in.
