US20250291730A1: 多芯粒处理系统中的硬件辅助页面迁移

MMU MCM 公开 2025-09-18
核心创新: 在每个芯粒(dielet)上设置硬件页面迁移引擎，通过 MMU 广播机制同步引擎页表绑定信息，利用访问计数器追踪远程访问并触发按需页面迁移。
技术解读:
  • 每个 dielet 上的硬件迁移引擎感知自身的页表绑定关系，可准确追踪跨 dielet 的内存流量
  • 通过 MMU 广播机制在多个 dielet 间同步绑定信息，确保访问计数的一致性
  • 支持基于访问计数的自动页面迁移决策，减少软件开销
  • 适用于通过 C2C/NVLink I/O 互连的多芯粒 MCM 架构
  • 解决了传统 GPU 页面迁移在多 dielet 场景下无法准确追踪远程流量的问题
影响分析: 该专利对 NVIDIA 多芯粒(MCM) GPU 架构至关重要。随着 Blackwell 及后续架构采用多 dielet 设计，硬件辅助的页面迁移可显著降低跨 dielet 内存访问延迟，提升 AI 大模型推理和训练的内存效率。
来源: Google Patents - US20250291730A1

US20250078199A1: 具有本地化模式的统一内存 GPU

MMU Unified Memory 公开 2025-03-06
核心创新: GPU 可选择性地将软件功能执行和相关数据存储资源限制在本地连接的处理/存储组件上，通过 MMU 页表项(PTE)中的属性位(attribute bit)实现内存访问的本地化或跨步(stride)映射。
技术解读:
  • MMU 使用 PTE 中的特殊属性位(AMAP bit)来标记内存页是本地还是远程
  • API 可将执行和数据绑定到特定的 uGPU，避免跨远程访问的延迟开销
  • 在统一内存架构下提供"本地模式"选项，兼顾灵活性和性能
  • 硬件级别强制执行本地化约束，无需软件额外检查
  • 适用于多 GPU / 多芯粒系统中减少远端内存访问延迟的场景
影响分析: 该专利为多芯粒 GPU 提供了关键的 MMU 级内存隔离机制。在 AI 推理场景中，可通过本地化模式显著降低 KV Cache 等热数据的访问延迟，同时保持统一内存的编程便利性。
来源: Google Patents - US20250078199A1

US20250307149A1: 用于缓存一致性的作用域树一致性协议

Cache Coherence Protocol 公开 2025-10-02
核心创新: 使用"作用域树"(scope tree)动态确定缓存层次结构中数据的一致性点(Point of Coherence, PoC)，支持在运行时根据处理器数量和位置迁移一致性点，减少总线流量。
技术解读:
  • 传统 GPU 缺乏硬件级缓存一致性，该协议通过作用域树结构实现动态一致性管理
  • 一致性点(PoC)可在缓存层次中迁移：当根缓存收到新请求时，从子缓存获取令牌(token)完成迁移
  • 相比传统嗅探(snooping)协议，大幅减少总线流量
  • 支持运行时根据处理器数量和位置动态调整一致性策略
  • 适用于多 GPU、多 SM 之间的缓存一致性管理
影响分析: 该专利解决了 GPU 缓存一致性的核心难题。在大规模多 GPU 训练和 DSMEM 场景中，高效的缓存一致性协议是确保数据正确性和性能的关键。该方案可能成为 NVIDIA 下一代 GPU 缓存一致性架构的基础。
来源: Google Patents - US20250307149A1

US20250291745A1: 自托管模式下网络接口与 GPU 的直接互连

Networking GPUDirect 公开 2025-09-18

技术解读:

  • 在"自托管模式"下，GPU 可同时作为计算和通信控制节点
  • 显著降低集合通信(如 NCCL AllReduce)的端到端延迟
  • 适用于大规模 GPU 集群中跨节点的高效数据传输
  • 与 GPUDirect RDMA 技术互补，进一步消除 CPU 在通信路径上的参与
影响分析: 该专利直接影响大规模 AI 训练集群的通信效率。在 10 万+ GPU 集群中，NIC-GPU 直接互连可显著减少集合通信的延迟和 CPU 开销，是 NVIDIA 网络-计算融合战略的关键专利。

US20230315655A1: 处理器和内存中的快速数据同步

Synchronization CGA 公开 2023-10-05
核心创新: 提出"光速"(Speed of Light, SOL)同步机制，通过组合的 store-and-arrive 操作实现约 0.5 个往返延迟的屏障同步，将生产者/消费者屏障分离以优化流水线。
技术解读:
  • SOL 同步将传统的屏障操作拆分为 store(存储)+ arrive(到达)两个组合操作
  • 延迟降低至约 0.5 个 RTT(往返时间)，接近理论极限
  • 分离的生产者/消费者屏障允许更灵活的流水线并行
  • 与 DSMEM 配合使用，可实现跨 SM 的高效数据交换
  • 是 CGA 编程模型中同步原语的硬件基础
影响分析: 该专利为 DSMEM 和 CGA 提供了底层同步机制。在大规模并行计算中，同步延迟是关键瓶颈，SOL 同步使得跨 SM 通信的开销大幅降低，直接提升了 DSMEM 的实用性。
来源: Google Patents - US20230315655A1

US20230393788A1: 封装内与封装外组合内存系统

  • 封装内内存(如 HBM)提供高带宽，封装外内存(如 DDR)提供大容量
  • 支持同时发起多种内存流量，提升总带宽利用率
  • 为 NVIDIA 的异构内存架构(如 Grace Hopper Superchip)提供专利保护
影响分析: 该专利覆盖了 NVIDIA Grace Hopper 等超级芯片的核心内存架构。在 AI 大模型推理中，统一的高带宽+大容量内存空间可支持更大的模型和更长的上下文窗口，是突破"内存墙"的关键技术。

计算通信并行研究论文

TokenWeave: 分布式 LLM 推理的高效计算-通信重叠

Microsoft Research MLSys 2026 Oral 2025-05
核心创新: 首个在 tensor parallelism 下实现小 token 长度(低至 1024)的高效计算-通信重叠系统。通过 Token-Splitting 技术将 AllReduce 通信与 GEMM 计算交织执行。
技术方案:
  • Token-Splitting 策略：将 token 维度切分为多个 micro-batch，通信和计算流水线式重叠
  • Fused AllReduce-RMSNorm 内核，利用 NVSHARP / Multimem 硬件原语
  • 无需修改模型结构或牺牲精度
  • 针对 NVLink 互联环境优化，通信开销从 20% 大幅降低
  • 已开源：GitHub - microsoft/tokenweave
来源: arXiv:2505.11329

Lagom: 释放分布式大模型训练中通信与计算重叠的潜力

中科大/NUS 2026-02 arXiv
核心创新: 提出联合调优通信参数以平衡通信与计算资源竞争的系统，使用统一代价模型将优化复杂度从指数级降至线性级。
技术方案:
  • 统一代价模型精确刻画 SM(流处理器)与带宽资源的竞争关系
  • 优先级指标 H 指导搜索最优通信参数配置
  • 优先级搜索算法将优化空间从指数级缩减到线性复杂度
  • 在高带宽集群上相比 NCCL 和 AutoCCL 实现 1.07x-1.33x 加速
  • 特别关注"计算成为瓶颈"场景下的重叠优化
来源: arXiv:2602.20656

FlashOverlap: 基于信号触发与重排序的高效计算-通信重叠

清华/无问芯穹 EuroSys 2026 2025-04
核心创新: 提出基于信号触发(signaling)的通信启动机制，在部分输出完成时即触发通信，同时通过重排序消除干扰，实现最高 1.65x 加速。
技术方案:
  • Signaling 机制：计算产出部分数据(tile 粒度)时即刻触发通信
  • Tile-wise 重叠：以 tile 为粒度实现计算与通信的细粒度交错
  • Wave-wise 分组：减少通信与计算之间的资源干扰
  • 执行顺序感知重排序：对 NCCL 内部实现透明(NCCL-agnostic)
  • 多种模型上实现最高 1.65x 端到端加速
来源: arXiv:2504.19519

NCCLX: 面向 10 万+ GPU 的集合通信框架

Meta 开源 2025-10 / 2026-01 修订
核心创新: Meta 开源的面向超大规模(10万+ GPU)集群的集合通信框架，统一支持训练和推理场景，在 Llama 4 模型上验证了显著通信效率提升。
技术方案:
  • CTran 传输层：实现 SM-free(不占用流处理器)的零拷贝 GPU 通信

  • 已在 Meta 生产环境部署并开源

Characterizing Compute-Communication Overlap: GPU 重叠特性深度刻画

Georgia Tech/KAIST IEEE 2025-07

  • 分析了不同消息大小、计算密度下的最优策略选择
  • 为"何时应该重叠、何时不应该"提供了量化指导
  • 功耗分析揭示重叠可能导致能效下降

  • 与 FlashAttention 等计算内核协同设计

  • 几乎不牺牲模型精度(near-lossless)

趋势展望

多芯粒(MCM)架构成为主流

NVIDIA 2-dielet Blackwell、AMD 8-XCD MI300X、华为 4-chip 昇腾 910D、Intel 2-tile Battlemage——所有主要厂商都已转向多芯粒设计。MMU 架构、缓存一致性、页面迁移成为多芯粒 GPU 的核心技术战场。

内存-计算融合加速

NIC-GPU 直连(US20250291745A1)、FAM 结构附加内存(US11822491B2)、DSMEM 分布式共享内存(US12248788B2)等专利都在模糊计算与内存的边界。处理器上堆叠内存(US20230275068A1)和垂直堆叠 1-2TB(US20250201793A1)代表了更激进的融合方向。

Token 粒度切分成为重叠新范式

TokenWeave (Microsoft) 和 FlashOverlap (清华) 都采用细粒度切分(token-splitting / tile-wise)实现计算通信交织，取代了传统的 layer-level 重叠策略。这代表了 2025-2026 年计算通信并行领域的核心趋势。

重叠并非总是有益 -- 需要智能策略

Georgia Tech/KAIST 的刻画研究揭示了重叠平均带来 18.9% 计算减速的重要发现。这意味着简单的"总是重叠"策略并不最优，未来需要基于工作负载特征和硬件状态的自适应重叠决策。

超大规模集群通信成为新战场

Meta NCCLX 面向 10万+ GPU、华为灵衢支持 15488 卡超节点——超大规模集群的通信效率成为新的技术竞争焦点。SM-free 通信(CTran)、容错集合操作、动态负载均衡是关键技术方向。

开放标准 vs 私有协议博弈加剧

UALink 1.0 (AMD+Intel+Google+Microsoft)、华为灵衢开源、海光 HSL 全栈开放，形成对 NVIDIA NVLink 私有协议的包围。开放标准能否在性能和生态上追平 NVLink 将是未来 2-3 年的关键看点。

国产 AI 芯片互连百花齐放

华为四芯片封装 + 灵衢 15488 卡超节点、寒武纪 MLU-Link 量产、壁仞光互连商用、海光 HSL 全栈开放。国产 AI 芯片在互连技术上正快速缩小与国际巨头的差距，互连生态的构建将成为下一阶段竞争焦点。

自动化通信优化成为趋势

Lagom 的统一代价模型、CCLInsight 的原语级分析、Intel 的自动流水线调度——从"手工调参"向"自动化通信优化"的演进趋势明显。未来的通信库将具备自适应调参能力，根据硬件拓扑和工作负载自动选择最优策略。
