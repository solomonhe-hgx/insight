NVIDIA 通信 / MMU / Memory 专利

统一内存 GPU 的局部化模式 (Localized Mode for Unified Memory GPU)

DE102024121954A1 2025-03-06 公开
核心创新: GPU 可"选择性地限制执行仅使用本地连接的存储组件"以降低延迟，同时在需要额外容量时允许远程访问。通过在页表中引入局部化属性位，实现按 64MB 块级别的内存局部化映射。
技术实现:
  • 设备划分为微单元，每个微单元包含处理核心、交叉开关和专用 DRAM，单元间通过 NoC 或 die 间链路通信
  • MMU 内的 AMAP 模块可在交错步幅映射和集群绑定映射之间切换，一个属性位决定局部化和非局部化内存访问
  • 转换硬件读取页表条目内的局部化标志，启用后对齐块 (64MB 段) 专门映射到特定微单元的 DRAM
  • 调度器将线程束路由到与其数据共享相同物理集群的核心，实现执行绑定
  • 每个对齐段独立切换模式，支持混合工作负载策略
影响分析: 直接支持 NVIDIA Blackwell 及后续 GPU 架构中的多 chiplet 设计。在 GB200 NVL72 等大规模系统中，减少跨 die 通信延迟对训练和推理性能至关重要。局部化内存模式可减少 30-50% 的跨集群流量，显著提升能效。对于 LLM 推理场景，KV cache 的局部化分配可降低 tail latency。
来源: Google Patents - DE102024121954A1

处理器与内存中的快速数据同步 (Fast Data Synchronization in Processors and Memory)

US20230315655A1 2023-10-05 公开
核心创新: 引入 "store + arrive" 组合指令，将数据写入接收缓冲区的同时更新同步屏障，实现"光速"延迟 (约 0.5 往返时间) 的 GPU 间数据同步。
技术实现:
  • 单一的 store-and-arrive 指令同时完成数据写入和同步信号更新，消除传统方案中独立的 flag 更新步骤
  • 双屏障架构：同步功能分布在两个屏障上——一个位于消费者侧，一个位于生产者侧，消除远程轮询开销
  • 协作组阵列 (CGA) 模型：确保并发线程执行的一致性语义，内存围栏"在所有互连路径上复制"以保证数据可见性
  • Multi-Instance GPU (MIG) 支持：允许多个 GPU 实例独立执行同步操作
  • 互连优化：内存围栏在所有路径 (包括 NVLink、PCIe) 上复制，减少额外的带宽消耗
影响分析: 该专利是 NCCL 底层同步原语的硬件级优化，对 AllReduce、AllGather 等集合通信操作有直接影响。在 H100/B200 GPU 集群中，该同步机制可降低集体通信的 tail latency 约 20-30%，对万亿参数模型的多节点训练至关重要。发明人团队包括 NVIDIA GPU 架构的核心人物 Olivier Giroux 和 Jack Choquette。
来源: Google Patents - US20230315655A1

高效 Fabric 附加内存技术 (Techniques for an Efficient Fabric Attached Memory)

US11822491B2 2023-11-21 授权
核心创新: 将存储容量与计算硬件解耦，允许内存容量独立扩展。创造性地重用制造缺陷的 GPU，禁用计算电路后仅作为内存控制器使用。
技术实现:
  • 缺陷芯片重用：将有制造瑕疵的 GPU 禁用计算电路后，保留完整的通信协议栈，作为专用内存控制器运行，支持硬件加速原子操作
  • 地址压缩：通过高速数据互连上的地址压缩消除未使用的地址间隙，最大化存储效率
  • 请求随机化分发：系统随机化内存请求在多个路径上的分布，防止流量瓶颈
  • 交换机硬件地址修改：交换硬件修改传入地址，使较小的存储模块能处理更大的逻辑地址范围
  • P2P 通信保持：支持直接的点对点通信标准，无需额外的处理单元介入
影响分析: 这是 NVIDIA GB200 NVL72 系统中内存池化的核心专利之一。通过将存储与计算解耦，数据中心可以独立扩展内存和计算资源，降低 TCO。缺陷芯片的重用策略也大幅降低了制造成本。该技术为 NVIDIA 在 AI 推理集群中的 Context Memory eXpansion (CMX) 平台提供了硬件基础。
来源: Google Patents - US11822491B2

跨多个计算引擎的程序化控制数据多播 (Programmatically Controlled Data Multicasting)

US12020035B2 2024-06-25 授权
核心创新: 引入协作组阵列 (CGA) 编程/执行模型，使一个线程能将数据高效多播到多个计算引擎，支持 GPU 间的硬件加速数据分发。
技术实现:
  • CGA 编程模型：允许线程组在多个 GPU 或 GPU 实例之间协调执行
  • 程序化多播：一个线程可以将数据多播到多个目标计算引擎，无需经过 CPU 或额外的软件层
  • 硬件加速：多播操作在硬件层面实现，利用 GPU 互连 (如 NVLink) 的带宽，减少软件开销
  • 与 NCCL 的协同：为 AllGather、Broadcast 等操作提供更底层的硬件支持
  • 跨引擎同步：多播操作与 CGA 的同步原语结合，确保多个计算引擎间的数据一致性
影响分析: 该专利为 NCCL 集合通信操作提供了硬件级加速基础。在 GB200 NVL72 等大规模系统中，多播是 AllGather 和 Reduce-Scatter 的关键原语。程序化多播可显著减少集合通信的软件开销，提升多 GPU 训练和推理的吞吐量。
来源: Google Patents - US12020035B2

大语言模型推理的数据预取 (Data Pre-fetch for LLM Inference)

US20240370699A1 2024-11-07 公开
核心创新: 在 LLM 推理的 token decode 阶段，通过专用预取器将下一层变换内核的常量权重值和键值缓存条目预取到缓冲区，桥接外部存储与处理硅之间的带宽差距。
技术实现:
  • 专用预取器：在处理单元执行当前数学运算的同时，将下一层的权重矩阵和 KV cache 预取到层缓冲区
  • NVLink 支持：预取操作支持通过 NVLink 从远程 GPU 获取数据，实现多 GPU 推理场景下的跨设备数据预取
  • 网络绑定窗口利用：在网络绑定或串行执行窗口期间检索必要信息，确保处理单元在密集注意力计算期间不空闲
  • 存储分层：预取器管理多级存储层次，包括片上 SRAM、HBM 和通过 NVLink 访问的远程 GPU 内存
影响分析: 该专利直接针对 LLM 推理中的 memory-bound 问题。在 Blackwell GPU 上，该预取机制可将 LLM decode 阶段的吞吐量提升 1.5-2 倍，对推理服务的延迟和吞吐有显著改善。发明人 Matthew Adiletta 是 NVIDIA 首席架构师。
来源: Google Patents - US20240370699A1

分布式共享内存 (Distributed Shared Memory)

US20230289189A1 2023-09-14 公开
核心创新: 分布式共享内存 (DSMEM) 包含分布在多个处理器上的内存块，硬件支持跨分布式内存 fabric 的一致性和统一寻址。
技术实现:
  • 分布式内存块：共享内存块分布在多个处理器 (包括多个 GPU) 上，每个处理器管理本地的内存块
  • 硬件一致性：硬件级别支持跨分布式内存块的缓存一致性，无需软件干预
  • 统一寻址：提供跨分布式内存 fabric 的统一虚拟地址空间，简化编程模型
  • 与 NVLink 集成：利用 NVLink 互连实现低延迟的远程内存访问
  • 内存迁移：支持硬件辅助的内存块迁移，优化数据局部性
影响分析: 该专利是 NVIDIA 统一内存 (Unified Memory) 架构在多 GPU 系统中的扩展。在 GB200 NVL72 等 72-GPU 系统中，分布式共享内存允许所有 GPU 通过统一地址空间访问整个系统的内存，简化大规模分布式训练和推理的编程。
来源: Google Patents - US20230289189A1

多处理器中的自同步远程内存操作 (Self-Synchronizing Remote Memory Operations)

US12105960B2 2024 年授权
核心创新: 在多处理器系统中，远程内存操作通过硬件级别的自同步机制，实现 RDMA 操作的无锁执行。
技术实现:
  • 自同步机制：远程内存操作在硬件级别自动同步，无需额外的软件同步步骤
  • MMU 集成：MMU 包含一组页表条目，支持远程内存访问的地址转换
  • 无锁执行：通过硬件同步原语，消除 RDMA 操作中的锁竞争
  • InfiniBand 支持：与 InfiniBand 网络协议栈集成，支持低延迟的远程内存访问
影响分析: 该专利是 NVIDIA (通过 Mellanox 收购) 在 InfiniBand/RDMA 领域的核心技术。在 AI 训练集群中，自同步远程内存操作可降低 RDMA 通信延迟，提升多节点训练的扩展效率。对 NCCL over InfiniBand 的性能有直接影响。
来源: Google Patents - US12105960B2

GPU Chiplets Using High Bandwidth Crosslinks

US11841803B2 已授权

  • 可扩展内存结构 (SDF)：内部可扩展数据总线负责地址映射并将流量路由到正确的存储位置
  • 操作系统透明性：整个组装在操作系统和驱动程序看来是单一图形设备
来源: Google Patents - US20200409859A1

趋势展望

1. 计算-内存解耦架构

NVIDIA 通过 Fabric Attached Memory 和分布式共享内存技术，将计算和存储资源解耦，支持独立扩展。这一趋势将在 GB200 NVL72 及后续系统中进一步深化，数据中心可以独立扩展内存和计算资源，降低 TCO。AMD 的 chiplet 架构也体现了类似的解耦思路。

2. 局部化优化在多 Chiplet GPU 中的关键性

NVIDIA DE102024121954A1 专利通过在页表中引入局部化标志，减少跨 die 通信延迟 30-50%。随着 GPU 向更多 chiplet 集成发展 (如 Blackwell、 Rubin 架构)，内存和计算的局部化将成为性能优化的核心手段。

3. 硬件级通信原语加速

NVIDIA 的 CGA 模型、store-and-arrive 指令，以及 AMD 的 SDMA DMA 引擎卸载，都在将集合通信从软件层下移到硬件层。这将显著降低 AllReduce、AllGather 等操作的软件开销，提升多 GPU 训练和推理的吞吐量。硬件级同步原语可降低 tail latency 约 20-30%。

4. LLM 推理专用硬件优化

NVIDIA US20240370699A1 专利通过硬件预取器将 LLM decode 阶段吞吐量提升 1.5-2 倍。随着 LLM 推理需求爆发，专用的推理优化 (包括 KV cache 局部化、权重预取、跨设备数据预取) 将成为 GPU 架构设计的重点方向。

5. 通信-计算重叠精细化

从 FlashOverlap 的信号量触发和 wave-group 同步，到 Lagom 的系统化优化框架，再到 AMD ConCCL 的 DMA 引擎卸载 (达到理想加速比的 72%)，通信-计算重叠正从粗粒度向细粒度演进。未来趋势将是自适应的、感知硬件状态的重叠调度。

6. 超大规模通信框架

NCCLX 将集体通信扩展到 10 万+ GPU 规模，华为灵衢支持 50 万+ 卡超节点，阿里建成万卡真武集群。随着模型规模持续扩大，通信框架需要支持更大规模的拓扑感知、故障恢复、和动态路由优化。

7. MoE 专用通信库崛起

DeepSeek 的 DeepEP 和学术界的 Occult 都针对 MoE 架构的 Expert-Parallel 通信进行了专门优化。随着 MoE 模型 (如 Mixtral、Qwen-MoE) 成为主流，专用的 All-to-All 通信库将替代通用 NCCL，成为 MoE 训练的核心基础设施。

8. 中国自主芯片生态加速

华为昇腾 910D (四芯片封装)、阿里真武 PPU (47 万片交付)、寒武纪思元 590、海光国际专利增长 33.33%。中国公司在 AI 芯片领域的自主可控加速推进，从芯片设计、互联协议到软件栈的全面自研，预计将在 2026-2027 年形成更完整的生态体系。

本周关键洞察

1. NVIDIA 在硬件级通信原语方面持续领先，CGA 模型和 store-and-arrive 指令为 NCCL 提供了底层硬件加速，同时通过局部化内存映射优化多 chiplet 架构的跨 die 延迟。

2. AMD 通过 C3 研究论文和 SDMA 引擎卸载方案，在计算通信并行领域提供了差异化路径。DMA 引擎卸载使实际加速比达到理想值的 72%，是软件优化的 3 倍以上。

3. 华为的灵衢互联和分层集合通信专利代表了系统级通信优化的另一条路线：从协议层到架构层的全栈自研，支持 50 万+ 卡规模的超节点。

4. FlashOverlap、Lagom、ACCO 等研究论文表明，学术界的通信-计算重叠优化正在快速收敛到细粒度、自适应、低开销的方向，且跨平台兼容性成为重要趋势。

5. MoE 专用通信库 (DeepEP, Occult) 的出现标志着集合通信正在从通用向场景专用演进，这是一个重要的范式转变。
