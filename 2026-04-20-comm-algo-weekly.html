● NCCL EP 论文发布 - MoE 通信统一 API 论文

2026-03 发布 / 本周持续讨论 ▼

核心内容

  • NCCL EP：基于 NCCL Device API 从头构建的 MoE 通信库，统一支持低延迟 (LL) 和高吞吐 (HT) 路径 arXiv 论文 ↗
  • 与 DeepEP 对比：统一 ncclEpDispatch / ncclEpCombine 原语，替代 DeepEP 的多接口分离设计；用 NCCL 原生拓扑检测替换 NVSHMEM 依赖 arXiv 论文 ↗
  • 性能表现：LL 设计实现较 DeepEP 架构一个数量级的显存占用降低；多节点 dispatch 吞吐匹配或超越 DeepEP；vLLM 端到端 serving 测试中，1-4 节点场景下性能为 DeepEP 的 90-93% arXiv 论文 ↗
  • 生态意义：解决当前 MoE 通信生态碎片化问题（DeepEP、pplx-kernels 等各自为战），为 NVIDIA 平台提供官方支持的 Expert Parallelism 路径 arXiv 论文 ↗ 中文解读 ↗

趋势展望

🔥 MoE 通信生态走向统一

NCCL EP 论文的发布标志着 NVIDIA 开始正视 MoE 通信碎片化问题。NCCL EP 以统一 API 替代 DeepEP 的多接口设计，并在 vLLM serving 中达到 DeepEP 90-93% 的性能。随着 AMD 同步完成 DeepEP ROCm 移植，MoE expert parallelism 的通信标准竞争将在 2026 Q2 进入白热化阶段。NCCL 能否在 Q2 roadmap 中将 EP 功能正式合并到主线，将是关键观察点。

NCCL EP 论文 ↗ ROCm/DeepEP ↗ NCCL Q2 Roadmap ↗

⚡ Device-Initiated 通信成为标配

本周 NCCL v2.30.3-1 的 GIN API 增强和 UCX v1.20 的 GPU device API 发布，加上 DeepEP 原生的 device-initiated all-to-all，标志着 GPU 通信正在从 CPU-proxy 模式全面转向 GPU 直接通信。NVSHMEM 的 tile API 和 device-side tensor construction 也沿着同一方向推进。这一趋势的核心驱动力是降低通信延迟和减少 CPU 瓶颈。

NCCL GIN API ↗ UCX GPU API ↗ NVSHMEM tile API ↗

🐍 Python 生态全面升级

NCCL v2.30.3-1 新增 freethreading 支持，NVSHMEM4Py 0.3.0 引入 CuTe DSL 和 device-side 构建，UCX 添加 Java API preview。所有主要通信库都在加大 Python 生态投入，这与 AI/ML 框架的 Python-first 趋势高度一致。torchcomms 的发布更是 PyTorch 对通信 API 简化的直接响应。

NCCL Python ↗ NVSHMEM4Py ↗ torchcomms ↗

🏗️ Blackwell / B200 适配加速

NCCL v2.30.3-1 的 NVLSTree Blackwell 调优、NVSHMEM 3.6.5 的 B200/GB200 GA 支持，以及 SGLang 社区在 B200 上的 NVSHMEM 实测，表明 2026 Q2 将是 Blackwell 通信栈成熟的关键季度。随着 Blackwell 产能爬坡，通信库的 Blackwell 优化将成为性能竞争的主战场。

NCCL Blackwell ↗ NVSHMEM B200 ↗ SGLang B200 测试 ↗

🌐 多网卡 / 多路径成为标配

NCCL v2.30.3-1 的 Dynamic Direct Path、NVSHMEM 3.6.5 的 libfabric Multi-NIC、UCX 的 Direct NIC 数据路径，所有主要通信库都在本周/近期更新中强化了多网卡支持。这反映了大规模 AI 集群对网络冗余和带宽聚合的刚性需求增长。

NCCL Direct Path ↗ NVSHMEM Multi-NIC ↗ UCX Direct NIC ↗

📌 本周观察

本周通信库竞争格局的核心变化是：NVIDIA 通过 NCCL v2.30.3-1 和 NCCL EP 论文发起 MoE 通信标准化攻势，直接对标 DeepSeek 的 DeepEP；同时 AMD 完成 DeepEP ROCm 移植形成跨平台呼应。NCCL 在 Device API、Elastic Buffers 和 Cross-Clique 上的大幅推进表明其正从"集合通信库"向"全场景 GPU 通信平台"转型。NVSHMEM 3.6.5 和 UCX v1.20 的同期更新进一步夯实了 Device-Initiated 通信和多网卡支持的行业标准。整体而言，2026 Q2 将是 MoE 通信生态格局重塑的关键季度。
