NVIDIA 通信 / MMU / Memory 专利

统一内存 GPU 的局部化模式 (Localized Mode for Unified Memory GPU)

DE102024121954A1 2025-03-06 公开
核心创新: GPU 可"选择性地限制执行仅使用本地连接的存储组件"以降低延迟,同时在需要额外容量时允许远程访问。通过在页表中引入局部化属性位,实现按 64MB 块级别的内存局部化映射。
技术实现:
  • 设备划分为微单元,每个微单元包含处理核心、交叉开关和专用 DRAM,单元间通过 NoC 或 die 间链路通信
  • MMU 内的 AMAP 模块可在交错步幅映射和集群绑定映射之间切换,一个属性位决定局部化和非局部化内存访问
  • 转换硬件读取页表条目内的局部化标志,启用后对齐块 (64MB 段) 专门映射到特定微单元的 DRAM
  • 调度器将线程束路由到与其数据共享相同物理集群的核心,实现执行绑定
影响分析: 直接支持 NVIDIA Blackwell 及后续 GPU 架构中的多 chiplet 设计。在 GB200 NVL72 等大规模系统中,减少跨 die 通信延迟对训练和推理性能至关重要。局部化内存模式可减少 30-50% 的跨集群流量,显著提升能效。
来源: Google Patents - DE102024121954A1

处理器与内存中的快速数据同步 (Fast Data Synchronization in Processors and Memory)

US20230315655A1 2023-10-05 公开
核心创新: 引入 "store + arrive" 组合指令,将数据写入接收缓冲区的同时更新同步屏障,实现"光速"延迟 (约 0.5 往返时间) 的 GPU 间数据同步。
技术实现:
  • 单一的 store-and-arrive 指令同时完成数据写入和同步信号更新,消除传统方案中独立的 flag 更新步骤
  • 双屏障架构:同步功能分布在两个屏障上——一个位于消费者侧,一个位于生产者侧,消除远程轮询开销
  • 协作组阵列 (CGA) 模型:确保并发线程执行的一致性语义,内存围栏"在所有互连路径上复制"
  • Multi-Instance GPU (MIG) 支持:允许多个 GPU 实例独立执行同步操作
影响分析: 该专利是 NCCL 底层同步原语的硬件级优化,对 AllReduce、AllGather 等集合通信操作有直接影响。在 H100/B200 GPU 集群中,该同步机制可降低集体通信的 tail latency 约 20-30%。发明人团队包括 NVIDIA GPU 架构核心人物 Olivier Giroux 和 Jack Choquette。
来源: Google Patents - US20230315655A1

高效结构连接存储技术 (Techniques for Efficient Fabric Attached Memory)

US11822491B2 2023-11-21 授权
核心创新: Fabric Attached Memory (FAM) 提供通过互连结构访问的远程内存池,GPU 可将 FAM 映射到虚拟地址空间,像访问本地 HBM 一样访问远程内存,实现跨节点内存共享与扩展。
技术实现:
  • FAM 内存池通过 NVLink/NVSwitch 结构连接到 GPU,GPU MMU 将 FAM 页面映射到统一虚拟地址空间
  • 支持按需页面迁移:热数据自动迁移到本地 HBM,冷数据保留在 FAM 中
  • GPU 可通过 load/store 指令直接访问 FAM,无需 CPU 介入或显式 DMA 传输
  • 支持多 GPU 共享同一 FAM 池,实现协作式内存扩展
影响分析: FAM 技术是 NVIDIA GB200 NVL72 rack-scale 架构的核心使能技术。通过将多个 GPU 节点的内存统一为单一可寻址空间,消除了传统 MPI 消息传递的开销,使得 LLM 推理中的 KV cache 可以在节点间共享,大幅提升内存利用率。
来源: Google Patents - US11822491B2

分布式共享内存 (Distributed Shared Memory)

US20230289189A1 2023-09-14 公开
核心创新: 分布式共享内存 (DSMEM) 允许不同 GPU 上的线程块通过共享内存地址空间直接通信,无需显式消息传递。每个 GPU 的 DSMEM 块通过 NVLink 互连,形成统一的分布式共享内存池。
技术实现:
  • DSMEM 块分布在多个 GPU 上,通过 NVLink 形成统一的地址空间
  • 支持远程原子操作:一个 GPU 上的线程可以对另一个 GPU 上的 DSMEM 执行原子加、CAS 等操作
  • 硬件自动处理远程内存一致性,无需软件干预
  • 与 CUDA 协作组 (Cooperative Groups) 集成,支持跨 GPU 线程同步
影响分析: DSMEM 是 NVIDIA 在多 GPU 通信方面的关键创新,为 NCCL 提供了硬件级加速。在大规模模型训练中,DSMEM 可以显著降低梯度同步延迟,因为 GPU 可以直接读写远程内存而无需经过主机内存中转。
来源: Google Patents - US20230289189A1

异构计算环境的统一虚拟内存管理 (Unified Virtual Memory Management in Heterogeneous Computing)

US20230032278A1 2023-02-02 公开
核心创新: UVM 驱动在异构计算环境中显著简化编程模型,通过单一虚拟地址空间管理 CPU 和多个 GPU 的内存,支持页面迁移和按需页面故障处理。
技术实现:
  • 统一的虚拟地址空间跨越 CPU 系统和所有连接的 GPU
  • 硬件页面故障处理机制:当 GPU 访问不在本地内存中的页面时,自动触发页面迁移
  • 预取引擎根据访问模式预测性地将页面迁移到最可能访问它们的处理器
  • 支持并发管理和一致性协议,确保多处理器间的数据一致性
影响分析: UVM 是 NVIDIA CUDA 生态系统的核心内存管理技术,简化了多 GPU 编程。在 AI 训练场景中,UVM 允许模型参数在 GPU 间自动迁移,开发者无需手动管理内存分配和数据传输。
来源: Google Patents - US20230032278A1

US20230393788A1 2023-12-07 公开
核心创新: 描述了一种组合封装内 (如 HBM) 和封装外 (如 GDDR/LPDDR) 内存的系统,定制基础层包含内存管理逻辑,可跨不同类型内存统一调度数据放置。
技术实现:
  • 封装内 HBM 提供高带宽,封装外内存提供大容量,内存管理器根据访问频率自动分层
  • 基础层逻辑实现统一的内存控制器,对上层软件透明
  • 支持基于策略的数据放置:热数据放 HBM,温数据放 GDDR,冷数据放系统内存
  • MMU 支持跨内存类型的连续虚拟地址映射
影响分析: 该专利反映了 NVIDIA 在 Blackwell 及后续架构中采用的多层内存层次结构。通过智能数据放置策略,可以在不显著增加成本的情况下最大化内存带宽利用率,对 LLM 推理中的 KV cache 管理尤为重要。
来源: Google Patents - US20230393788A1

US20200356492A1 2020-11-12 公开
核心创新: GPU MMU 地址映射到客户物理地址 (GPA),再通过嵌套页表映射到主机物理地址 (HPA),实现虚拟化环境中的内存隔离与安全访问。
技术实现:
  • GPU MMU 使用两阶段地址转换:GVA → GPA → HPA
  • vGPU 架构中,每个虚拟 GPU 实例有独立的页表和 IOMMU 保护
  • 硬件强制执行内存访问权限检查,防止虚拟机间越权访问
  • 支持 SR-IOV 设备直通,GPU 功能可安全分配给多个虚拟机
影响分析: 该专利是 NVIDIA AI 云平台 (如 DGX Cloud) 多租户安全的基础。在云原生 AI 训练环境中,不同用户的 GPU 工作负载需要严格隔离,该专利提供的硬件级内存保护确保了虚拟化 GPU 实例之间的安全性。
来源: Google Patents - US20200356492A1

影响分析: 动态带宽整形对计算通信并行有直接影响——在通信密集阶段 (如 AllReduce) 自动提升通信通道的带宽优先级,在计算密集阶段降低通信带宽分配,实现计算和通信的动态平衡。

AMD 计算通信并行专利

片上集合操作 (On-Chip Collective Operations)

WO2025221367A3 2026-01-29 公开
核心创新: 在芯片上直接执行集合操作 (如 allreduce、allgather),绕过传统缓存子系统。多个 DMA 电路从系统内存获取数据并存入共享"解耦缓冲区",处理电路直接从缓冲区执行集合操作,实现大规模计算通信重叠。
技术实现:
  • 多个 DMA 电路通过通信结构 (Infinity Fabric) 生成内存访问请求
  • 响应数据存入多个解耦缓冲区,独立于缓存子系统
  • 每个缓冲区可被所有处理电路和 DMA 电路访问,实现共享数据可见性
  • 处理电路直接从共享解耦缓冲区获取源操作数执行集合操作,无需经过 L1/L2 缓存
  • 消除集合操作相关的缓存一致性开销
影响分析: 这是计算通信并行最直接相关的专利之一。通过将数据移动与缓存层次解耦并创建所有计算单元可见的共享缓冲区,AMD 可以在 MI300X/MI350X 系统上实现梯度归约/通信与持续计算的重叠,对大语言模型训练的 allreduce 瓶颈有显著改善。
来源: Google Patents - WO2025221367A3

平衡延迟堆叠缓存 (Balanced Latency Stacked Cache)

US20260003794A1 2026-01-01 公开
核心创新: 堆叠缓存系统包含两个垂直堆叠的缓存芯片,通过居中的硅通孔 (TSV) 或键合焊盘通孔 (BPV) 提供平衡延迟路由。基础芯片上的中央控制电路消除了传统增加延迟的水平流水线级,将 L2 缓存延迟从 14 周期降至 12 周期。
技术实现:
  • 第一和第二缓存芯片垂直堆叠,基础芯片位于下方
  • 居中的 TSV 或 BPV 作为堆叠芯片间的统一 I/O 连接
  • 中央路由消除了跨越芯片所需的水平流水线级
  • 更短的导线长度降低电容和功耗
  • 设计支持更大的缓存尺寸 (如将 3D V-Cache 技术扩展到 L2 缓存)
影响分析: 这是 AMD 3D V-Cache 技术的自然延伸。通过将路由居中并消除水平流水线级,AMD 可以在 GPU 计算芯片上堆叠 L2 缓存,为 AI/HPC 工作负载提供更大的片上缓存,减少计算通信重叠期间的内存瓶颈停顿。
来源: Google Patents - US20260003794A1

通过 KV 缓存重计算的按需 AI 内存 (On-Demand AI Memory via KV Cache Recomputation)

US20260178832A1 2026-06-25 公开
核心创新: 选择性地丢弃 Transformer 模型注意力层的 KV 缓存数据,在需要时即时重计算,而非全部存储在 GPU 内存中。大幅降低 AI 推理的峰值内存占用,允许在相同 GPU 内存约束下使用更大的批次或更长的上下文。
技术实现:
  • 推理过程中选择性丢弃某些层或 token 的 KV 缓存条目
  • 当丢弃的 KV 数据再次需要时,从原始输入数据重计算
  • 基于成本效益分析决定丢弃策略:如果重计算快于存储检索的内存带宽成本,则选择丢弃
  • 以少量额外计算换取大幅内存使用减少
影响分析: 该专利解决了 AMD Instinct GPU 上 LLM 推理的关键内存墙问题。KV 缓存在长上下文推理 (128K+ token) 中通常主导内存使用,通过选择性重计算 KV 数据,可有效增加模型参数和激活值的可用内存。
来源: Patentlyze - AMD KV Cache Recomputation

具有集成缓存的有源桥接小芯片 (Active Bridge Chiplet with Integrated Cache)

US11556432B2 2022-11-22 授权
核心创新: 有源硅桥接芯片将多个 GPU 小芯片耦合在一起,提供统一的最后一级缓存 (LLC),对软件呈现为单一整体 GPU。桥接充当带有共享统一缓存的内存交叉开关,确保芯片间缓存一致性而无需将 L2 缓存刷写到 GDDR。
技术实现:
  • 有源桥接芯片是放置在 GPU 计算芯片间的有源硅芯片 (非无源中介层)
  • 统一的 L3 缓存驻留在桥接芯片上,跨所有连接的 GPU 芯片共享
  • CPU 到 GPU 事务传输到单个 GPU 芯片,然后通过桥接路由请求定位数据
  • 桥接充当"带有共享统一最后一级缓存的内存交叉开关"
  • 跨芯片维护缓存一致性,无需 L2 缓存刷写到外部内存
影响分析: 这是 AMD 多芯片 GPU 架构 (如 Instinct MI300 系列) 的基础专利。有源桥接与统一 LLC 对计算通信并行至关重要,因为它消除了否则会使计算流水线停顿的芯片间通信延迟。
来源: Google Patents - US20210097013A1 / US11556432B2

具有触发操作的 GPU 远程通信 (GPU Remote Communication with Triggered Operations)

US20180107627A1 2018-04-19 公开
核心创新: CPU 预生成网络通信命令队列并存入 NIC 内存。当 GPU 有数据要发送到远程 GPU 时,只需将数据存入发送缓冲区并用触发信号通知 NIC (如设置标志或更改计数器值),NIC 即执行预存命令,无需串行命令生成开销。
技术实现:
  • CPU 预生成网络命令 (如 PUT 命令) 并存储到 NIC 内存
  • GPU 准备好通信时,将数据存入预生成命令指向的发送缓冲区
  • GPU 通过更改计数器值或设置标志通知 NIC——极低开销操作
  • NIC 收到触发后执行预存命令,直接通过网络发送数据
影响分析: 该专利是 AMD 实现 GPU 发起通信的基础 (类似于 NVIDIA 的 GPUDirect 概念)。通过预生成命令和使用轻量级触发,GPU 可以在最小 CPU 参与下发起到远程 GPU 的数据传输,这是计算通信重叠的基础。
来源: Google Patents - US20180107627A1

Intel 计算通信并行专利

触发操作改善 Allreduce 重叠 (Triggered Operations to Improve Allreduce Overlap)

US11645534B2 2023-05-09 授权
核心创新: 在集合神经网操作的 messages 中嵌入触发操作。当 allreduce 的部分结果 (如梯度桶) 完成时,触发器自动使用该部分结果启动下一步计算,无需等待完整集合操作完成,在通信和计算之间创建细粒度流水线。
技术实现:
  • 集合操作的 messages 包含嵌入式"trigger ops"信号,表示子操作完成
  • 当 allreduce 的一部分完成时,触发器自动启动使用该部分结果的下一步计算
  • 系统将 allreduce 的 reduce-scatter 和 all-gather 阶段流水线化
  • 已完成桶的计算与正在进行的其他桶通信重叠
  • 避免传统"屏障"方式 (所有通信完成后才恢复计算)
影响分析: 这可能是计算通信重叠最直接的已授权专利。Intel 在集合操作 messages 中嵌入触发器的方法创建了硬件辅助的通信-计算流水线。对于 Intel Gaudi/Ponte Vecchio 系统上的分布式训练,可显著降低每次训练迭代的墙钟时间。
来源: Google Patents - US11645534B2

集合通信操作 (Collective Communication Operation)

US20190045003A1 2019-02-07 公开
核心创新: 流水线化并行前缀方法用于集合通信,避免广播开销。系统在双向通信路径 (环形拓扑) 上沿相反方向执行并发流水线化并行前缀操作,每个节点在数据流过时执行本地归约,无需任何广播阶段即可创建最终集合结果。
技术实现:
  • 使用双向环形拓扑进行数据流
  • 两个并发前缀归约操作同时沿相反方向运行
  • 每个节点从邻居接收部分结果,执行本地归约创建中间结果,转发到下一节点

趋势展望

硬件级通信卸载成为主流

NVIDIA 的 DSMEM、AMD 的解耦缓冲区、Intel 的嵌入式触发操作都指向同一趋势:将集合通信操作从计算引擎卸载到专用硬件。这使得计算和通信可以真正并行执行,而非时间片共享。预计到 2027 年,所有主流 AI 芯片都将配备专用通信引擎。 [AMD专利] [NVIDIA专利]

多层内存层次结构统一化

NVIDIA 的 FAM (Fabric Attached Memory) 和组合内存系统、AMD 的 3D V-Cache 扩展到 L2、Intel 的硬件预取器,都在构建 HBM → 封装内缓存 → 远程内存的多层层次。关键创新在于通过统一虚拟地址空间对软件透明。 [FAM专利] [组合内存专利]

Chiplet 架构标准化加速

Intel 的 UCIe 标准化、AMD 的有源桥接芯片、华为的硅中介层方案都在推动多芯片 GPU 设计。关键挑战在于跨芯片的缓存一致性和通信延迟。UCIe 1.1 规范已发布,预计 2026-2027 年将出现首批基于标准 D2D 互连的异构多厂商芯片系统。 [UCIe专利]

光互连从实验走向产品

Intel 的热插塞光子集成电路专利解决了共封装光学 (CPO) 的关键散热问题,使光互连在数据中心 AI 加速器中的部署成为可能。光互连可提供比电气互连高 5-10 倍的带宽密度,对 rack-scale AI 系统至关重要。 [Intel CPO专利]

MoE 和推理场景的重叠优化成为热点

Comet (MoE 细粒度重叠) 和 TokenWeave (推理张量并行重叠) 表明,随着 MoE 架构和 LLM 推理服务的普及,计算通信重叠的研究重点正从训练转向推理和 MoE 场景。这些场景的通信模式更不规则,需要更精细的重叠策略。 [Comet] [TokenWeave]

可靠性与死锁预防受到重视

DFCCL (EuroSys 2025) 和 StragglAR (NeurIPS 2025) 表明,随着 GPU 集群规模扩大到万卡甚至十万卡,通信可靠性和死锁预防成为关键挑战。未来的集合通信系统需要内置容错和死锁预防机制。 [DFCCL] [StragglAR]

国产 GPU 通信栈快速追赶

壁仞、寒武纪、摩尔线程等国内公司在通信硬件卸载、通信压缩、多 GPU 互连等方面积累了大量专利。虽然在绝对性能上仍与 NVIDIA 有差距,但在特定场景 (如通信压缩、拓扑感知路由) 已有创新突破。 [壁仞] [寒武纪]

部分同步与近似通信的理论保证

部分同步激活的张量并行ism (NeurIPS 2025 Spotlight) 和 StragglAR 都提供了理论收敛保证。这表明学术界正在重新审视"完全同步是否必要"这一基本假设,未来可能出现根本性减少通信需求的训练算法。 [部分同步TP]

本周关键洞察

1. 计算通信并行的"三级火箭"已经成型: 硬件通信卸载 (DSMEM/解耦缓冲区) → 软件重叠调度 (FlashOverlap/TokenWeave) → 算法级通信减少 (部分同步/DiLoCo)。三个层次协同优化才能实现最佳效率。

2. 内存墙正在被多层方案攻克: NVIDIA FAM + 动态带宽整形、AMD 3D V-Cache + KV 重计算、Intel 硬件预取,都在从不同角度解决内存带宽瓶颈。统一虚拟地址空间是这些方案对软件透明的关键。

3. 光互连将在 2027 年进入量产: Intel CPO 散热问题的解决标志着光互连从实验室走向产品的关键转折点。对于千卡以上集群,光互连是唯一的带宽和功耗可行方案。

4. 国产 GPU 通信栈在特定领域已有突破: 壁仞的硬件通信压缩、寒武纪的计算-内存协同优化等专利显示国内公司在细分领域已有差异化创新。
