  • NVLink 分区通过 NVSwitch Control Plane 和 NVOS CLI 命令管理

架构创新

一致性内存池: CPU-GPU 统一地址空间

核心更新: NVLink-C2C 实现 CPU 和 GPU 内存的硬件一致性，使 LPDDR5X 和 HBM4 可作为单一一致性内存池访问。
技术解读:
  • 应用可将 CPU 和 GPU 内存视为统一池，无需显式数据移动
  • KV-cache offload 可利用 CPU 侧 LPDDR5X 扩展有效 GPU 内存容量
  • 多模型执行时，模型可在 CPU/GPU 内存间无缝迁移
  • 硬件一致性协议自动处理缓存同步，无需软件显式管理
  • 对 LLM 推理场景意义重大：减少数据移动开销，提升吞吐量
驱动影响:
  • R595 引入的 CDMM (Coherent Driver-Based Memory Management) 是软件层面的关键支撑
  • CDMM 由驱动管理 GPU 内存而非 OS，避免 GPU 内存作为 NUMA 节点暴露
  • 通过 NVreg_CoherentGPUMemoryMode=driver 内核参数启用
  • 注意: CDMM 模式下不支持 GPU Direct Storage
来源: NVIDIA Developer Blog | R595 Release Notes - CDMM

新特性

CDMM: Coherent Driver-Based Memory Management (R595)

核心更新: R595 驱动引入 CDMM，由驱动管理 GPU 内存而非操作系统，专为 GB200 平台设计。
技术详情:
  • CDMM 使驱动直接管理 GPU 内存，避免 OS 在线化 GPU 内存和将其暴露为 NUMA 节点
  • 启用方式: echo options nvidia NVreg_CoherentGPUMemoryMode=driver > /etc/modprobe.d/nvidia-openrm.conf
  • 适用于 GB200 平台的 NVLink-C2C 一致性内存架构
  • 限制: CDMM 模式下不支持 GPU Direct Storage
  • 对 open kernel modules 的影响: 需要通过 OpenRM 配置模块参数
代码路径: kernel-open/nvidia/ (模块参数), src/nvidia/ (内存管理逻辑)

已知问题

Blackwell TMA 描述符 MMU Fault (< 128KB 内存)

问题描述: 在 Blackwell GPU 上使用 cuTensorMapEncodeTiled() 时，若 tensor 内存分配小于 128KB，可能出现 sporadic IMA 或 MMU fault。
技术详情:
  • 驱动在配置 TMA 描述符时存在缺陷，当 tensor 的 backing memory 分配小于 128KB 时触发
  • 症状: 非法内存访问 (IMA) 或 MMU fault
  • Workaround: 创建描述符后清除 bit 85: ((uint64_t*)&tensorMap)[1] &= ~((uint64_t)1 << 21);
  • 影响所有 Blackwell GPU (compute capability 10.x)
代码路径: src/nvidia/ (TMA descriptor configuration), kernel-open/nvidia-uvm/ (MMU fault handling)
来源: NVIDIA R595 Release Notes - Known Issues
