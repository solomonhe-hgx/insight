NVIDIA 核心专利 (2025-2026)

US12536056B2: 硬件加速异步事务同步与分布式共享内存 (DSMEM)

Hardware Accelerated Synchronization with Asynchronous Transaction Support

NVIDIA Corp 申请: 2022-03-10 | 授权: 2026-01-27
核心创新: 提出"事务屏障"(transaction barrier) 机制，通过专用硬件单元同步线程与异步硬件事务，实现分布式共享内存 (DSMEM) 的高效访问。DSMEM 将各 SM 的共享内存块聚合为跨 GPU 的分布式共享内存空间。
技术解读:
  • 分布式共享内存 (DSMEM): 将同一 GPU 上各 Streaming Multiprocessor (SM) 的共享内存块聚合为统一的分布式共享内存空间，允许不同 SM 的线程直接访问彼此的共享内存
  • 事务屏障 (Transaction Barrier): 新型同步原语，将线程同步与异步硬件事务操作结合。当线程到达屏障时，触发异步事务（如原子操作、数据搬运），无需等待事务完成即可继续执行其他工作
  • 硬件加速: 专用硬件单元管理事务屏障，比软件同步方式显著降低延迟。支持 store-and-arrive 组合指令，将数据存储和屏障到达合并为单条指令
  • 快速同步协议: 实现约 0.5 个往返时间的同步延迟（传统方式需 1 个完整 RTT），通过远程屏障更新机制减少同步开销
  • 与 Blackwell 架构关系: DSMEM 在 Blackwell 架构中得到硬件级支持，每个 SM 拥有更大的共享内存，配合 NVLink 实现跨 chiplet 的内存共享
影响分析: 该专利是 NVIDIA GPU 片上通信基础设施的核心。DSMEM 使同一 GPU 内的 SM 间可以直接共享数据而无需经过全局内存，大幅降低通信延迟。配合 NVLink 5/6 的跨 GPU 内存一致性，形成完整的片上-片间通信层次结构。对 MoE 模型中 expert 间的数据交换尤为重要。
来源: Google Patents - US12536056B2, Google Patents - US20230289189A1 (公开版)

NVIDIA 动态 KV Cache 调度专利 (2025年申请)

Dynamic KV Cache Scheduling for Large Language Models

NVIDIA Corp 申请: 2025-02-13 | 公开: 2026-05-14
核心创新: 提出分层 KV Cache 层次结构（L1/L2 处理器内 -> 二级内存 -> 可丢弃层），基于使用频率和重新引用可能性的调度策略，实现 LLM 推理中 KV Cache 的智能内存管理。采用按需分页模型，仅在模型需要时将离线的 KV 对重新加载到快速 GPU 内存。
技术解读:
  • 分层 KV Cache 层次: 第一层为处理器内高速缓存 (L1/L2)，第二层为处理器外内存 (HBM/主机内存)，第三层为可丢弃层（可重新计算的数据）
  • 智能调度策略: 基于 recency (最近使用) 和 re-reference likelihood (重新引用可能性) 决定 KV 对在层次间的迁移方向
  • 按需分页 (Demand Paging): 类似操作系统虚拟内存的按需分页机制，当模型需要已离线的 KV 对时，触发页面故障并重新加载
  • 与 ICMS 协同: 该专利与 NVIDIA 的 Inference Context Memory Storage Platform (ICMS, BlueField-4 驱动) 形成软硬件协同方案，ICMS 提供第三层存储基础设施
  • 解决 LLM 推理内存瓶颈: 长上下文 LLM 推理中 KV Cache 可占 GPU 内存 60-80%，该专利通过智能调度大幅提升有效内存容量
影响分析: 该专利直接面向 LLM 推理的核心瓶颈 -- KV Cache 内存管理。分层调度策略可显著提升单 GPU 支持的并发请求数和上下文长度。结合 NIXL 推理传输库和 ICMS 存储平台，NVIDIA 构建了完整的分布式推理内存管理专利组合。
来源: Patentlyze - NVIDIA Dynamic KV Cache Scheduling, Semiconductor Engineering - HBM & KV Cache

US20230393788A1: 封装内与封装外组合内存系统

Combined On-Package and Off-Package Memory System

NVIDIA Corp 申请: 2023-08-23 | 公开: 2024-06
核心创新: 定制基础层将封装内 HBM 与封装外 DDR/LPDDR 整合为统一内存空间。逻辑层智能路由内存请求以优化带宽和容量，支持近/远内存池及跨池透明数据迁移。地址翻译和映射操作从 CPU 卸载到专用硬件。
技术解读:
  • 双内存池架构: 近内存池 (near memory) 为封装内 HBM，提供高带宽；远内存池 (far memory) 为封装外 DDR/LPDDR，提供大容量
  • 统一内存空间: 对上层软件呈现为单一连续地址空间，无需应用感知底层内存层次
  • 智能请求路由: 硬件逻辑根据访问模式动态将请求路由到 HBM (带宽优先) 或 DDR (容量优先)
  • 透明数据迁移: 热数据自动从远内存迁移到近内存，冷数据反向迁移，类似 CPU 缓存层次但粒度更大
  • 硬件地址翻译: 专用硬件执行地址翻译和映射，减轻 CPU/GPU 的地址管理负担
  • 与 Grace Hopper 关系: 该专利是 Grace Hopper Superchip 统一内存架构的专利基础，Grace CPU 的 LPDDR5X 与 GPU 的 HBM 构成近/远内存池
影响分析: 该专利定义了 NVIDIA 未来 GPU 内存架构的基本范式。随着 Rubin 采用 HBM4 (288GB) + NVLink-C2C 连接 Grace CPU 的 LPDDR，这种组合内存系统将成为标准配置。对大模型训练中内存容量受限的问题提供了系统性解决方案。
来源: Google Patents - US20230393788A1

US20230315655A1: 处理器与内存中的快速数据同步

Fast Data Synchronization in Processors and Memory

NVIDIA Corp 申请: 2022-03-10 | 公开: 2023-10
核心创新: 提出组合 store-and-arrive 指令和分离式生产者/消费者屏障机制，通过远程屏障更新将同步延迟降低至约 0.5 个往返时间 (RTT)，相比传统两阶段同步减少约 50% 延迟。
技术解读:
  • Store-and-Arrive 组合指令: 将数据写入 (store) 和屏障到达 (arrive) 合并为单条硬件指令，消除两条独立指令间的流水线气泡
  • 分离式屏障 (Split Barrier): 生产者线程和消费者线程使用独立的屏障控制，允许生产者提前写入数据而不必等待消费者就绪
  • 远程屏障更新: 屏障状态更新可直接在远端完成，无需回合到发起端，将延迟从 1 RTT 降至 0.5 RTT
  • SOL (Speed-of-Light) 同步: 接近物理极限的同步速度，对片上网络 (NoC) 和 NVLink 互连同适用
  • 与 DSMEM 协同: 该同步机制是分布式共享内存 (DSMEM) 高效运行的基础，确保多 SM 间的数据一致性
影响分析: 快速数据同步是 GPU 并行计算效率的基石。0.5 RTT 的同步延迟接近物理极限，对大规模 GPU 集群中的 collective communication (如 AllReduce, AllGather) 性能至关重要。该专利与 US12536056B2 共同构成 NVIDIA GPU 片上同步的专利壁垒。
来源: Google Patents - US20230315655A1

US11182309B2: 高效 Fabric 附加内存技术

Techniques for an Efficient Fabric Attached Memory

NVIDIA Corp 申请: 2019-11-04 | 授权: 2021-11-23
核心创新: 将内存从计算资源中解耦，使用高速互连实现独立的内存扩展。创新性地利用低能力 GPU 作为低成本 Fabric 附加内存控制器，通过地址压缩和熵基链路选择 (spraying) 优化跨互连的内存访问效率。
技术解读:
  • 内存解耦架构: 将内存资源从 GPU 计算节点分离，通过高速 fabric 互连提供远程内存访问，实现内存容量的独立扩展
  • GPU 作为内存控制器: 创造性地复用低能力 GPU (或降级 GPU) 作为 Fabric 附加内存的控制器，降低额外硬件成本
  • 地址压缩: 在高速数据互连上实现地址压缩，充分利用 FAM 容量，解决地址空间不匹配问题
  • 熵基链路选择 (Entropy-based Spraying): 基于熵的负载均衡算法，在多条互连链路间分散请求，防止热点
  • 硬件加速原子操作: 在 donor GPU 上支持原生硬件原子操作，无需软件模拟
  • 与 NVIDIA MGX/HGX 关系: 该专利是 NVIDIA 数据中心内存解耦战略的基础，影响后续 BlueField DPU 和 ICMS 架构设计
影响分析: Fabric Attached Memory 是 NVIDIA 内存解耦战略的专利基础。随着 LLM 模型规模增长，单节点内存容量成为瓶颈，FAM 提供了不增加计算资源即可扩展内存的方案。该专利对理解 NVIDIA ICMS (Inference Context Memory Storage) 架构有重要参考价值。

US20230126350A1: 非易失性内存存储与接口

Non-Volatile Memory Storage and Interface

NVIDIA Corp 申请: 2021-10-25 | 公开: 2023-04-27
核心创新: 提出虚拟内存块 (VMB) 技术替代传统非易失性内存 (NVM) 功能，无需修改关联控制器。通过 GPU 链路接口实现高速非易失性存储访问，为 AI 推理提供持久化的 KV Cache 和模型权重存储。
技术解读:
  • VMB (Virtual Memory Block): 用虚拟化方式抽象非易失性存储，使上层软件无需感知底层存储介质变化
  • GPU 链路接口: 通过 GPU 专用链路 (如 NVLink) 直接访问非易失性存储，绕过 PCIe 瓶颈
  • 控制器兼容性: VMB 可替代一个或多个 NVM 功能，无需修改现有控制器设计
  • 与 ICMS 平台关系: 该专利是 NVIDIA ICMS (Inference Context Memory Storage) 平台的技术基础之一，ICMS 利用 BlueField-4 DPU 实现 GPU 到持久化存储的高速访问
影响分析: 该专利为 NVIDIA 在 AI 推理持久化内存领域奠定基础。结合 Dynamic KV Cache Scheduling 专利，NVIDIA 构建了从 GPU HBM -> 主机内存 -> 非易失性存储的完整推理内存层次。

HBM4 内存控制器与统一虚拟内存 (UVM) 优化

HBM4 / UVM ISSCC 2026 / arXiv:2507.02770
核心创新: NVIDIA 在 ISSCC 2026 展示 HBM4 集成方案，结合 UVM 软件页表遍历优化。Rubin 平台将搭载 288GB HBM4 (每 GPU)，带宽超过 2TB/s。软件页表遍历支持不规则 GPU 应用，显著减少 page fault 开销。
技术解读:

  • 软件页表遍历: 支持 GPU 上的软件页表遍历 (software page table walk)，针对不规则内存访问模式优化，减少硬件 TLB miss 时的延迟惩罚
  • UVM 增强: 统一虚拟内存 (UVM) 支持跨 CPU-GPU 的透明页面迁移，基于访问模式的自动数据放置
  • 页面故障处理: 从 Pascal 架构延续并增强的页面故障处理能力，支持 GPU 直接发起页面迁移请求
  • Rubin 内存系统: 每机架内存容量达 20.7TB (72 GPU x 288GB)，配合 NVLink 6 实现机架级统一内存池
影响分析: HBM4 与 UVM 的结合使 GPU 内存管理更加高效。288GB 单 GPU 内存容量可容纳更大的模型和 batch size，对万亿参数模型训练至关重要。软件页表遍历的引入表明 GPU 内存管理正从纯硬件方案向软硬件协同方向演进。

Vera Rubin 平台: 下一代内存架构 (GTC 2026)

Vera Rubin GTC 2026 发布
核心创新: NVIDIA Vera Rubin 平台采用七芯片协同设计: R100 GPU (3nm, 336B 晶体管) + Vera CPU (227B 晶体管) + NVLink 6 + HBM4。单 GPU 288GB HBM4，机架级 20.7TB 内存，NVLink 6 双向带宽 3.6TB/s per GPU。
技术解读:
  • R100 GPU: TSMC 3nm 工艺，336B 晶体管，相比 Blackwell 的 208B 晶体管增长 62%
  • HBM4 内存: 每 GPU 288GB HBM4，带宽超过 2TB/s，机架级 20.7TB (72 GPU)
  • NVLink 6: 第六代 NVLink 互连，每 GPU 双向带宽 3.6TB/s，支持 72 GPU 全互连拓扑
  • Vera CPU: 自研 ARM 架构 CPU，227B 晶体管，通过 NVLink-C2C 与 GPU 连接
  • 七芯片协同: GPU + CPU + NVSwitch + BlueField-4 + ConnectX-8 + ICMS +散热管理芯片
  • 内存带宽效率: NVLink 6 使 72 GPU 的聚合内存带宽达 1.5PB/s (72 x 20.7TB/s)
影响分析: Vera Rubin 代表了 NVIDIA 在 AI 硬件领域的全面飞跃。336B 晶体管和 288GB HBM4 使其成为有史以来最大的商用芯片。机架级 20.7TB 统一内存可容纳万亿参数模型而无需跨机架通信。

影响分析: NCCL 2.26 系列修复了大规模多节点部署中的关键稳定性问题。MNNVL alltoall hang 修复对 GB300 NVL72 等多 GPU 系统至关重要。

NIXL: NVIDIA 推理传输库 (Inference Transfer Library)

NIXL (开源) GTC 2026 发布
核心创新: NIXL 是 NVIDIA 面向分布式推理的开源库，专为 KV cache 高效传输设计。支持预填-解码分离架构中的 KV block 快速迁移，利用 RDMA 和零拷贝传输最小化处理延迟。
技术解读:
  • 中央协调器 + 分布式运行时代理架构，每个代理包含内存注册表和元数据管理器
  • 统一 API 暴露"可插拔后端插件"，跨多样网络和存储层路由操作
  • 标准化描述符覆盖主机 RAM、GPU 内存和持久存储
  • 默认基于 UCX 协议，动态元数据交换建立 peer-to-peer 通信通道
  • KV cache 管理: 在预填和 decode worker 间快速迁移 KV blocks
影响分析: NIXL 是 NVIDIA 在 disaggregated inference 领域的关键基础设施。随着 LLM 推理从单节点向分布式架构演进，KV cache 的高效传输成为核心瓶颈。NIXL 类似于训练领域的 NCCL，旨在成为推理通信的标准库。

NCCLbpf: 可验证可组合的 GPU 集体通信策略执行

arXiv:2603.11438 2026-03 发布
核心创新: 将 userspace eBPF 运行时嵌入 NCCL 的插件接口，替换未验证的原生插件，消除"作业崩溃、静默状态损坏和停机"风险。AllReduce 吞吐量在 4-128 MiB 范围提升最高 27%。
技术解读:
  • 直接嵌入 NCCL 现有插件接口，无需修改 NCCL 本身
  • 加载时静态验证防止不安全插件执行，测试中成功阻止危险行为
  • 通过共享内存结构支持可组合策略和闭环自适应
  • 原子级策略热重载 (hot-reload)，无需停机
  • 消息大小感知的调优，在 B300 硬件上每次调优决策仅增加 80-130 ns 开销
影响分析: NCCLbpf 解决了 NCCL 插件安全性的关键问题。27% 的 AllReduce 提升对大规模训练有显著影响，且开销仅 80-130 ns。eBPF 沙箱化为大规模集群的通信可靠性提供保障。

NVIDIA MMU / 地址翻译技术 (2025-2026)

LATPC: 局部性感知 TLB 预取与 MSHR 压缩加速 GPU 地址翻译

Locality-Aware TLB Prefetching and MSHR Compression

ISCA 2025 ACM ISCA 2025
核心创新: LATPC 结合局部性感知的 TLB 预取和 MSHR (Miss Status Holding Register) 压缩技术，加速 GPU 地址翻译。通过预测性 TLB 预取减少页表遍历 (page table walk) 延迟，同时通过 MSHR 压缩提高并发地址翻译效率。
技术解读:
  • TLB 预取: 利用 GPU 内存访问的局部性模式，提前将可能需要的页表项加载到 TLB 中，减少 TLB miss 时的页表遍历开销
  • 局部性感知: 分析 GPU 工作负载的空间和 temporal 局部性特征，自适应调整预取策略
  • MSHR 压缩: 合并具有相同页表遍历路径的多个 miss 请求，减少并发页表遍历的数量和内存带宽消耗
  • 与 GPU MMU 层次的关系: NVIDIA GPU 采用多级 TLB 层次 (L1 TLB -> L2 TLB -> 页表)，LATPC 优化了 L1/L2 TLB 的预取和 miss 处理
  • 对 UVM 的影响: 在统一虚拟内存 (UVM) 场景下，地址翻译延迟更高 (需要跨 CPU-GPU 页表同步)，LATPC 的预取机制可显著降低此类开销
影响分析: LATPC 代表了 GPU 地址翻译优化的最新学术进展。随着 GPU 工作负载越来越多样化 (图处理、GNN、稀疏注意力等不规则访问模式增多)，TLB miss 开销占比上升。LATPC 的预取和压缩技术对 NVIDIA 未来 GPU 的 MMU 设计有重要参考价值。
来源: ACM DL - LATPC, ResearchGate - LATPC

NVIDIA GPU MMU 演进: 从 UVM 到 NVLink-C2C 硬件地址翻译

MMU 架构演进 Pascal -> Blackwell -> Rubin
核心创新: NVIDIA GPU MMU 经历了从基础虚拟内存 -> UVM (统一虚拟内存) -> UM (统一内存) -> NVLink-C2C HAT (硬件地址翻译) 的完整演进。每代架构在页表结构、TLB 层次和页面迁移机制上持续增强。
技术解读:
  • Pascal (2016): 首次引入页面故障处理 (page fault handling)，支持 49-bit 虚拟地址，4 级页表
  • Volta/Turing (2017-2018): 引入 UVM (Unified Virtual Memory)，支持 CPU-GPU 透明页面迁移
  • Ampere (2020): 6 级页表支持，更大的 TLB 层次，UVM 性能优化
  • Hopper (2022): 引入 NVLink-C2C HAT (Hardware Address Translation)，Grace CPU 与 Hopper GPU 间硬件地址翻译，消除软件翻译开销
  • Blackwell (2024-2025): 软件页表遍历支持，DSMEM 地址翻译，NVLink 5 跨 GPU 地址一致性
  • Rubin (2026): 预计进一步增强 HAT，支持 NVLink 6 的 72 GPU 统一虚拟地址空间，288GB HBM4 的页表管理
  • 关键专利支撑: US8531471B2 (共享虚拟内存)、US20140281299A1 (UVM 页面迁移)、US20230393788A1 (组合内存系统)
影响分析: NVIDIA GPU MMU 的演进路线清晰地展示了从简单虚拟内存向完整统一内存架构的发展方向。HAT (硬件地址翻译) 的引入是关键转折点，使跨 chiplet 的内存访问延迟接近片上访问。这对理解 NVIDIA 的内存一致性模型和编程模型演进至关重要。
来源: 知乎 - NVIDIA GPU MMU 演进, Google Patents - US8531471B2 (共享虚拟内存), Google Patents - US20140281299A1 (UVM 页面迁移)

GPUVM: GPU 驱动的统一虚拟内存

GPU-driven Unified Virtual Memory

arXiv:2411.05309 2024-11 发布
核心创新: GPUVM 提出 GPU 驱动的按需分页方案，将页面迁移决策从 CPU 移到 GPU，减少 CPU-GPU 间的页面迁移延迟。GPU 直接管理虚拟到物理地址翻译和页表遍历，无需 CPU 干预。
技术解读:
  • GPU 驱动分页: 传统 UVM 中页面迁移由 CPU 驱动，GPUVM 将迁移决策移到 GPU，减少 CPU-GPU 通信开销
  • 按需分页: GPU 在需要时直接发起页面迁移请求，无需等待 CPU 调度
  • 地址翻译优化: GPU 端直接管理页表，减少跨 CPU-GPU 的页表遍历
  • 与 NVIDIA UVM 的关系: GPUVM 的研究方向与 NVIDIA 从 UVM 到 UM 的演进一致，Blackwell/Rubin 中软件页表遍历支持体现了类似思路
影响分析: GPUVM 代表了学术界对 GPU 内存管理未来方向的探索。将页面管理从 CPU 移到 GPU 的趋势与 NVIDIA 硬件演进一致 (HAT, 软件页表遍历)，对理解 GPU MMU 的未来发展有参考价值。
来源: arXiv:2411.05309 - GPUVM, ResearchGate - GPUVM

趋势展望

GPU MMU 向软硬件协同演进

从 Pascal 的纯硬件页面故障处理到 Blackwell 的软件页表遍历支持，NVIDIA GPU MMU 正从纯硬件方案向软硬件协同方向演进。LATPC 等学术研究进一步探索 TLB 预取和 MSHR 压缩。HAT (硬件地址翻译) 的引入使跨 chiplet 地址翻译接近片上速度。LATPC, GPU MMU 演进

内存层次从 3 层扩展到 6 层

NVIDIA 的内存层次已从传统的 L1/L2/HBM 三层扩展到六层: L1 -> L2 -> HBM4 -> NVLink-C2C (CPU LPDDR) -> NVLink (跨 GPU) -> ICMS (持久化存储)。Dynamic KV Cache Scheduling 专利正是为管理这一复杂层次而设计。KV Cache 调度专利, ICMS

NVLink 开放生态战略加速

NVLink Fusion 开放第三方 IP 授权，NVLink-C2C 消除封装级延迟，NVLink 5/6 实现机架级全互连。NVIDIA 正将 NVLink 从封闭互连转变为平台级生态，与 UCIe 标准形成直接竞争。734 项美国专利 (2026 Patent 300) 构建了强大的专利壁垒。NVLink Fusion, Patent 300

推理内存管理成为新战场

Dynamic KV Cache Scheduling + NIXL + ICMS 构成 NVIDIA 完整的推理内存管理方案。LLM 推理中 KV Cache 可占 GPU 内存 60-80%，智能调度可显著提升并发和上下文长度。预填-解码分离架构成为标准部署模式。NIXL, Hybrid EP

通信安全性与可验证性

NCCLbpf 将 eBPF 沙箱引入 GPU 集体通信，消除原生插件安全风险。27% AllReduce 提升证明安全性与性能可以兼得。随着 GPU 集群规模增长 (数万 GPU)，通信可靠性成为关键需求。NCCLbpf

内存解耦与 Fabric Attached Memory

US11182309B2 专利的 FAM 概念正在成为现实: ICMS 将持久化存储解耦为独立资源层，BlueField-4 DPU 管理数据传输。内存扩展不再依赖计算节点增长，降低 TCO。这对云服务商的 AI 基础设施规划有重要影响。FAM 专利, ICMS Blog
