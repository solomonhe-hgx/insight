US12361628B2: 可配置多芯片 GPU 架构

AMD Chiplet Multi-Die GPU 互连 授权 2025-07-15 | 申请 2022-12-08
核心创新: 将 GPU 拆分为多个芯片(chiplet)，支持动态切换"统一模式"(作为单个 GPU 运行)和"分区模式"(作为多个独立 GPU 运行)，通过前端引擎动态分配管理不同 shader 引擎子集。
技术解读:
  • GPU 被分区为多个 die，每个 die 包含独立的 shader 引擎和前端控制逻辑
  • 统一模式下，单一前端引擎统一调度所有 die 上的 shader 引擎，对外表现为单个 GPU
  • 分区模式下，多个前端引擎分别管理各自的 shader 引擎子集，可独立处理不同任务
  • 支持混合组装(mixed binning)：不同品质等级的 die 可组合在同一封装中
  • 对 Infinity Fabric 互连带宽和一致性协议提出了新的设计要求
  • 为多 GPU 通信模式(如 RCCL 集合通信)提供了硬件层面的灵活性
影响分析: 该专利是 AMD 在 GPU chiplet 领域的核心布局。可配置的多模式 GPU 架构使 AMD 能够灵活应对 AI 训练(需要大规模统一内存)和推理(可分区处理独立请求)的不同需求。这一设计直接影响 GPU 间通信模式，为计算通信并行提供了硬件基础。与 NVIDIA 的 NVSwitch 方案相比，AMD 选择了更灵活的 chiplet 可配置路线。
来源: Google Patents - US12361628B2

US20260003794A1: 平衡延迟堆叠缓存 (Balanced Latency Stacked Cache)

AMD Cache 3D 堆叠 内存层次 公开 2026-01-01 | 申请 2024-06-28
核心创新: 提出一种堆叠缓存系统，包含第一和第二缓存 die、中央控制电路和中心连接过孔(center connection vias)，通过中央路由实现均衡的访问延迟，避免传统堆叠方案中边缘 die 访问延迟过高的问题。
技术解读:
  • 多层缓存 die 通过 3D 堆叠方式集成，中心控制电路位于堆叠结构中央
  • 中心连接过孔(via)提供从中央到各层缓存 die 的均匀访问路径
  • 消除了传统方案中因物理距离差异导致的延迟不均衡问题
  • 对 Infinity Cache 技术进行了 3D 扩展，可显著提升缓存命中率和访问效率
  • 适用于 AI 推理和训练场景中大规模 L3/LLC 缓存的管理
  • 与 HBM 内存配合，可在 GPU 封装内实现多级缓存层次结构
影响分析: 该专利代表了 AMD 在 3D 缓存堆叠领域的重要进展。平衡延迟设计解决了多层缓存架构中的关键瓶颈——随着堆叠层数增加，边缘缓存的访问延迟显著上升。这一技术对 AMD 下一代 Instinct 加速器的缓存架构至关重要，直接影响 GPU 计算单元的数据供给效率，进而影响计算通信重叠的效果。
来源: Google Patents - US20260003794A1

US12080632B2: 玻璃核心封装基板 (Glass Core Package Substrates)

AMD 先进封装 玻璃基板 互连 授权 2024-09-03 (2025年仍活跃) | 申请 2021-09-29
核心创新: 使用玻璃基板替代传统有机封装材料，通过两个玻璃基板直接键合(无气隙/底部填充)实现高密度互连，显著提升信号完整性和热管理能力。
技术解读:
  • 玻璃基板相比有机基板具有更低介电常数和损耗，支持更高频率信号传输
  • 双玻璃基板直接键合消除了传统封装中的气隙和底部填充材料
  • 单侧重布线层(redistribution layer)简化了制造工艺
  • 玻璃基板的热膨胀系数(CTE)更稳定，减少热应力导致的翘曲
  • 为 Infinity Fabric 互连提供了更高质量的物理层基础
  • 可支持更大规模的 chiplet 封装，对多 GPU/多 die 设计至关重要
影响分析: 玻璃基板被视为下一代芯片封装的革命性技术。AMD 的这一专利为其未来大规模 GPU chiplet 设计提供了关键的封装基础。相比 Intel 在玻璃基板上的投入，AMD 的方案更侧重于双基板键合的简化制造路径。这一技术直接影响 GPU 互连的信号质量和带宽密度，是支撑高速 Infinity Fabric 通信的物理层关键。
来源: Google Patents - US12080632B2

US11841803B2: 高带宽交叉互连的 GPU Chiplet

AMD Chiplet 交叉互连 一致性缓存 授权 2023-12-12 (2025年持续引用) | 申请 2019-06-28
核心创新: 多个 GPU chiplet 通过被动交叉互连(passive crosslink) die 连接，形成统一的一致性最后一级缓存(L3)，对外表现为单个单体 GPU。CPU 通过总线与第一 GPU chiplet 通信，其余 GPU chiplet 通过交叉互连 die 桥接。
技术解读:
  • 被动交叉互连 die 是一个无源硅中介层，连接多个有源 GPU chiplet
  • 跨 chiplet 的统一 LLC 实现一致性内存访问，简化软件编程模型
  • CPU 通过高速总线(如 Infinity Fabric)直接连接主 GPU chiplet
  • 可扩展架构：增加 GPU chiplet 数量即可提升计算能力
  • 制造良率优势：小 chiplet 比大单体 die 更容易保证良率
  • 为 AMD CDNA 架构的多 die 设计奠定了基础
影响分析: 这是 AMD GPU chiplet 架构的奠基性专利，直接影响了 RDNA 3 和 CDNA 3/4 的设计。高带宽交叉互连方案解决了多 GPU die 之间的通信瓶颈，为集合通信操作(如 AllReduce)提供了硬件加速路径。该专利在 2025 年仍被大量后续专利引用，是理解 AMD 多 GPU 互连策略的关键。
来源: Google Patents - US11841803B2

US20210097013A1: 集成缓存的有源桥接 Chiplet

AMD 有源桥接 一致性缓存 SoC 拆分 公开 2021-04-01 (2025年持续引用) | 申请 2019-09-27
核心创新: 使用有源硅 die 作为桥接 chiplet，内置统一最后一级缓存(LLC)，连接多个 GPU chiplet。有源桥接 die 不仅传递信号，还承担缓存一致性管理功能，使多个 GPU chiplet 对软件表现为单个单体 GPU。
技术解读:
  • 有源桥接 chiplet 是有源硅 die(区别于被动中介层)，内含 LLC 缓存
  • LLC 缓存在所有连接的 GPU chiplet 之间保持一致性
  • 支持将 SoC 功能拆分为更小的功能 chiplet 组合
  • CPU 通过总线连接第一 GPU chiplet，GPU chiplet 之间通过有源桥接通信
  • 桥接 die 的 L2 缓存在单个 GPU chiplet 内部保持一致
  • 为 AMD Infinity Fabric 互连协议提供了缓存一致性支持
影响分析: 该专利是 AMD 有源桥接 chiplet 技术的核心文献，在 2025 年仍被广泛引用。与被动交叉互连方案相比，有源桥接提供了更智能的缓存管理和一致性支持。这一设计直接影响了 AMD Instinct MI300 系列的 3D chiplet 架构，对多 GPU 间的计算通信并行具有深远影响。
来源: Google Patents - US20210097013A1

US12493922B2: GPU 处理与缓存改进

Intel GPU Cache 功耗优化 带宽管理 授权 2025-12-09 | 申请 2023-10-19
核心创新: 通过优化 IP 核之间的缓存数据流管理，使用流式缓冲区(streaming buffers)减少 GPU 处理中的功耗、延迟，同时提升带宽利用率。针对 GPU 缓存层次结构中的冗余数据移动进行了系统性优化。
技术解读:
  • 在 GPU 的多个 IP 核之间引入优化的流式缓冲机制
  • 减少缓存行在 L1/L2/L3 层次间的无效迁移
  • 通过预取策略和缓存替换算法降低内存访问延迟
  • 优化功耗管理：在低利用率场景下动态降低缓存功耗
  • 适用于 Intel Arc/Battlemage GPU 架构中的 Xe 核心集群
  • 对 oneAPI Level Zero 编程模型中的内存管理有直接影响
影响分析: 该专利是 Intel 在 GPU 缓存优化领域的最新布局，直接影响 Intel Arc 和数据中心 GPU 产品的性能表现。在 AI 推理和训练场景中，高效的缓存管理可减少 GPU 计算单元的空闲等待时间，提升计算通信重叠效率。与 NVIDIA 的硬件级组播内存方案不同，Intel 选择了缓存流式优化的路线。
来源: Google Patents - US12493922B2

US12347783B2: 硅中介层与 EMIB 互连架构

Intel EMIB 硅中介层 互连 授权 2025-07-01 | 申请 2024-01-05
核心创新: 将硅中介层(interposer)与 EMIB(嵌入式多芯片互连桥)结合，interposer 连接高密度 die cube(FPGA+内存)，EMIB 连接传统 die 到 interposer，在提升带宽的同时改善良率和成本。
技术解读:
  • 封装基板上方放置硅中介层(interposer)，提供高密度布线通道
  • die cube(如 FPGA + HBM 内存堆叠)通过 interposer 实现高带宽互连
  • EMIB 作为局部互连桥，将传统 die(legacy die)连接到 interposer
  • EMIB 仅在有源区域需要高密度互连时使用，降低成本
  • interposer 支持 die cube 之间的高速数据通路
  • 该架构可应用于 Intel Xe Link 互连和 oneAPI 的多设备通信
影响分析: 该专利是 Intel 先进封装战略的核心。Interposer + EMIB 的混合互连方案为 Intel 的多 die GPU 和加速器设计提供了灵活的物理层基础。在数据中心场景中，这一架构支持 FPGA + GPU + 内存的异构集成，对 oneAPI 编程模型下的跨设备通信和计算通信并行具有重要影响。
来源: Google Patents - US12347783B2

US12112398B2: SoC 架构解耦 (Disaggregation of SOC Architecture)

Intel SoC 解耦 Chiplet 互连 Fabric 授权 2024-10-08 (2025年持续引用) | 申请 2023-09-20
核心创新: 将 SoC 架构解耦为模块化 chiplet，支持通过互连 fabric 交换不同的图形/内存单元，实现灵活的工作负载分配和可定制的硬件配置。
技术解读:
  • SoC 被拆分为多个独立的 chiplet 模块：计算 tile、内存 tile、I/O tile 等
  • 各 chiplet 通过高速互连 fabric(如 UCIe)进行通信
  • 支持动态替换和升级单个 chiplet，无需重新设计整个 SoC
  • 内存控制器 chiplet 可独立配置，支持 HBM、DDR 等不同内存类型
  • 图形处理 chiplet 可按需增减，灵活适配不同性能等级
  • 为 Intel 的 Xe 架构 GPU 提供了模块化设计基础
影响分析: 该专利代表了 Intel 在 GPU SoC 解耦方面的核心思路。通过模块化 chiplet 设计，Intel 可以快速推出不同性能等级的 GPU 产品，同时降低设计和制造成本。这一架构对 Intel 数据中心 GPU(如 Max 系列)的多设备通信模式有直接影响，是 oneAPI 跨设备编程模型的硬件基础。
来源: Google Patents - US12112398B2

US11663135B2: 互连 Fabric 中的偏置一致性

Intel 缓存一致性 互连 Fabric 内存偏置 授权 2023-05-30 (2025年持续引用) | 申请 2021-12-20
核心创新: 在互连 fabric 中实现动态偏置一致性(bias-based coherency)，内存访问可在"主机偏置"(通过 CPU 一致性域)和"设备偏置"(直接绕过一致性协议)之间切换，优化带宽和功耗。
技术解读:
  • 加速器本地内存支持两种访问模式：host bias 和 device bias
  • host bias 模式：内存访问通过 CPU 的一致性域，保证数据一致性
  • device bias 模式：加速器直接访问本地内存，绕过一致性开销
  • fabric 控制器根据工作负载特征动态切换偏置模式
  • 减少了不必要的缓存一致性流量，降低互连 fabric 的拥塞
  • 适用于 CXL/PCIe 互连场景下的 CPU-GPU 内存共享
影响分析: 该专利解决了异构计算系统中 CPU-GPU 内存一致性的核心难题。动态偏置切换使系统可根据工作负载特征(计算密集型 vs 通信密集型)自动优化内存访问路径。在计算通信并行场景中，这一机制可显著减少一致性协议带来的通信开销，是 Intel oneAPI 内存模型的关键硬件支撑。
来源: Google Patents - US11663135B2

US12231304B2: 服务等级合规的高效资源分配

Intel 资源分配 SLA 计算通信调度 授权 2025-02-18 | 申请 2020-12-21
核心创新: 使用硬件级 SLO(如每指令周期数 CPI)作为性能指标，通过机器学习预测工作负载的抖动(jitter)，利用贝叶斯优化自动配置微服务的缓存和内存带宽分配，确保 SLA 合规的同时最小化资源过度配置。
技术解读:
  • 采用 CPI(Cycles Per Instruction)作为硬件级性能监控指标
  • 机器学习模型分析工作负载特征，预测性能抖动和尾延迟
  • 贝叶斯优化自动搜索最优的缓存/带宽分配配置
  • 动态调整内存带宽分配(MBA)和 L3 缓存分配(CAT)
  • 确保微服务在共享硬件环境中的 SLA 合规性
  • 对数据中心 GPU 集群中的计算通信资源调度有直接参考价值
影响分析: 该专利为数据中心级别的资源管理提供了智能化方案。在大规模 GPU 训练集群中，计算和通信资源的动态分配直接影响训练效率和尾延迟。Intel 的方案将硬件级性能监控与 ML 驱动的自动调优相结合，为 oneAPI 编程模型下的计算通信并行优化提供了系统级支持。
来源: Google Patents - US12231304B2

US20240370699A1: 大语言模型处理的数据预取

Intel LLM 数据预取 计算通信重叠 公开 2024-11-07 | 申请 2024-07-05
核心创新: 在 LLM 推理/训练过程中，利用网络同步等低带宽阶段，将下一层的常量权重和 KV cache 数据预取到缓冲区，消除计算引擎需要高带宽时的等待延迟，实现计算与通信的有效重叠。
技术解读:
  • 针对 Transformer 架构的 LLM 处理进行了专门优化
  • 识别出 LLM 处理中的"低带宽阶段"(如 AllReduce 集合通信期间)
  • 在低带宽阶段，利用空闲的内存带宽预取下一层的权重和 KV cache
  • 当计算引擎进入需要高带宽的阶段时，数据已就绪，无需等待
  • 硬件电路支持常量权重和键值对的分阶段预取
  • 直接适用于分布式 LLM 训练中的计算-通信重叠优化
影响分析: 该专利直接针对分布式 LLM 训练中的核心性能瓶颈——计算与通信的同步等待问题。通过在通信阶段预取计算数据，Intel 提出了一种硬件级的计算通信重叠方案。相比软件层面的流水线调度，硬件预取可以更精确地匹配带宽利用模式，对 Intel Gaudi 加速器和 oneAPI 编程模型中的分布式训练效率有直接影响。
来源: Google Patents - US20240370699A1

趋势展望

Chiplet GPU 成为主流

AMD 和 Intel 均在积极布局 GPU chiplet 架构。AMD 的可配置多模式 GPU (US12361628B2) 和 Intel 的 SoC 解耦方案 (US12112398B2) 表明，chiplet 已从 CPU 扩展到 GPU/加速器领域。未来 2-3 年，大规模多 die GPU 将成为 AI 训练集群的标准配置。

3D 堆叠缓存竞争加剧

AMD 的平衡延迟堆叠缓存 (US20260003794A1) 与 Intel 的 GPU 缓存优化 (US12493922B2) 反映了两家在缓存架构上的不同策略。AMD 侧重 3D 物理堆叠提升容量，Intel 侧重数据流优化提升效率。两种路线将在未来 AI 加速器中持续竞争。

计算通信重叠成为核心竞争力

Intel 的 LLM 数据预取专利 (US20240370699A1) 明确将计算通信重叠作为硬件设计目标。随着模型规模增长，通信开销占比持续上升，硬件级的计算通信重叠将成为加速器竞争的关键差异化因素。Meta 开源的 RCCLX 也印证了这一趋势。

开放互连标准 UALink/UCIe 加速

AMD、Intel、Google 等联合推动 UALink 标准，对抗 NVIDIA NVLink 的封闭生态。Intel 的偏置一致性专利 (US11663135B2) 和 EMIB 互连方案 (US12347783B2) 为开放互连标准提供了技术基础。未来加速器互连将向标准化、开放化方向发展。

玻璃基板封装竞赛

AMD (US12080632B2) 和 Intel 均在布局玻璃基板技术。相比有机基板，玻璃基板支持更高密度的互连和更大的封装尺寸，是下一代超大 chiplet GPU 的关键使能技术。预计 2027-2028 年将进入量产阶段。

AI 驱动的自适应资源管理

Intel 的智能资源分配专利 (US12231304B2) 展示了 ML 驱动的自适应硬件配置趋势。未来加速器将根据工作负载特征自动调整计算/通信/缓存资源分配，实现真正的"硬件感知"分布式训练。这与 AMD ROCm 7.0 的软件优化方向互补。
