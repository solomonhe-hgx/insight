NVIDIA 通信 / MMU / Memory 专利

分布式共享内存 (Distributed Shared Memory - DSMEM)

US20230289189A1 2023-09-14 公开
核心创新: 硬件和软件框架实现多个 SM 直接访问彼此本地内存池,创建分布式共享内存系统,绕过 L2 缓存实现低延迟跨 SM 通信。
技术实现:
  • 协作组阵列 (CGA) 层次化编程模型,统一工作分发器支持推测性 CGA 启动
  • 通用内存地址空间包含全局内存和映射的 DSMEM 本地内存窗口
  • 跨 SM 访问使用 CGA 内逻辑 CTA ID 定位特定内存区域
  • 内容可寻址路由表将逻辑 ID 翻译为物理 SM 目标,互连包编码直接 SM-to-SM 加载/存储/原子操作
  • 每个 CGA 专用硬件屏障强制执行同步,MEMBAR 指令确保严格读写排序
  • GPC Manager 跟踪活动线程计数,仅在确认所有参与 SM 的 DSMEM 内存刷新完成后才释放 CGA ID
影响分析: 该专利是 NVIDIA 在硬件级分布式共享内存的基础性工作,支撑 Hopper 及以后 GPU 架构中的协作组编程模型。在 GB200 NVL72 等多 GPU 系统中,DSMEM 允许所有 GPU 通过统一地址空间访问整个系统内存,简化大规模分布式训练和推理。
来源: Google Patents - US20230289189A1

高效 Fabric 附加内存技术 (Techniques for an Efficient Fabric Attached Memory)

US11822491B2 2023-11-21 授权
核心创新: 将存储容量与计算硬件解耦,允许内存容量独立扩展。创造性地重用制造缺陷的 GPU 作为内存控制器。
技术实现:
  • 缺陷芯片重用:将有制造瑕疵的 GPU 禁用计算电路后,保留完整通信协议栈,作为专用内存控制器运行,支持硬件加速原子操作
  • 地址压缩:通过高速数据互连上的地址压缩消除未使用的地址间隙,最大化存储效率
  • 熵驱动"Swizzle"路由:内存请求在多个互连链路上随机分发,防止流量拥塞
  • 交换机硬件地址修改:传入地址被修改以适配较小存储模块的线性地址窗口
  • 内存条带化:跨多个 FAMM 控制器条带化实现并行带宽扩展,支持多租户虚拟化隔离
  • 模块化硬件设计:FAMM 实现为可互换机架基板,数据中心可按需物理交换计算托盘和内存托盘
影响分析: 这是 NVIDIA GB200 NVL72 系统中内存池化的核心专利之一。通过解耦存储与计算,数据中心可以独立扩展内存和计算资源,降低 TCO。缺陷芯片重用策略大幅降低制造成本,为 Context Memory eXpansion (CMX) 平台提供硬件基础。
来源: Google Patents - US11822491B2

CCCL: GPU 内压缩耦合集合通信 (CCCL: In-GPU Compression-Coupled Collective Communication)

arXiv:2604.17172 2026-04 发布
核心创新: 将压缩内核紧密融合到 NCCL 数据处理层,利用 GPU 空闲资源执行压缩,减少通信量同时保持计算效率。
技术实现:
  • GPU 原生压缩引擎与本地频率表配对,直接融合到 NCCL 数据处理层
  • Warp 级执行,无需用户代码修改
  • 压缩与集合通信操作同步执行,利用 GPU 上未被使用的计算资源
  • 支持点对点 (P2P) 和集体通信模式
影响分析: CCCL 代表了集合通信的新方向:通过 GPU 内压缩减少通信量,在大模型训练中可显著降低 AllReduce/AllGather 的网络带宽需求。这是对 NCCL 架构的重要补充,可能成为未来 NVIDIA GPU 通信库的标准特性。
来源: arXiv:2604.17172, PDF

SoftWalker: 支持不规则 GPU 应用的软件页表遍历 (SoftWalker: Supporting Software Page Table Walk for Irregular GPU Applications)

ACM DOI:10.1145/3725843.3756056 2025 发表
核心创新: 提出可扩展的软件页表遍历方案 SoftWalker,解决 GPU MMU 在处理不规则内存访问模式时的 TLB miss contention 问题。
技术实现:
  • 软件页表遍历器替代硬件 Page Table Walker,减少 SM 间 TLB miss 竞争
  • 在真实 NVIDIA GPU 上评估页表遍历 contention
  • 对图计算、稀疏矩阵等不规则访问模式有显著加速
  • 4 KiB 最小页粒度支持
影响分析: SoftWalker 针对 GPU MMU 在处理不规则内存访问时的性能瓶颈提出了解决方案。在 LLM 推理中,KV cache 的稀疏访问模式可能从此优化中受益。
来源: ACM Digital Library

C3: 优化 ML 并发计算与通信 — GPU DMA 引擎加速

arXiv:2412.14335 AMD 研究论文
核心贡献: 提出将通信任务卸载到 GPU DMA 引擎,构建并发通信集合 (ConCCL) 概念验证,平均达到理想加速比的 72%,最高达 1.67 倍加速。
技术要点:
  • C3 平均仅达到理想加速比的 21%,由于并发 GPU kernel 之间在 CU 和 HBM/缓存上的干扰
  • 调度优先级和计算单元资源分区策略将性能提升至 42%
  • 将通信任务卸载到 GPU DMA 引擎后,达到理想加速比的 72%
  • 实验平台:AMD Instinct MI300X (304 CUs, 14 SDMA engines, 192GB HBM)
影响分析: C3 研究揭示了 GPU 上计算-通信并行的核心挑战:共享资源 (CU、HBM、缓存) 的竞争。通过 DMA 引擎卸载,AMD 提供了一条不同于 NVIDIA Copy Engine Collectives 的路径。MI300X 上的 14 个 SDMA 引擎为通信提供了专用硬件资源。
来源: arXiv:2412.14335

GPU Chiplets Using High Bandwidth Crosslinks & Active Bridge

US11841803B2 / US20210097013A1 已授权
核心创新: 将 GPU 拆分为多个 chiplet 并通过高带宽互连连接,有源桥接芯片集成 L3 缓存作为路由交叉开关。
技术实现:
  • 被动中介层:使用 "passive interposer die" 管理跨芯片数据路由,无 TSV
  • 有源硅桥接:作为路由交叉开关运行,托管共享 L3 层级,维护完整内存一致性
  • 统一缓存一致性:在所有物理芯片之间维护单一一致的 LLC
  • 可扩展内存结构 (SDF):内部总线负责地址映射和流量路由
  • 操作系统透明性:组合硬件对软件表现为单体 GPU
影响分析: 该专利是 AMD MI300 系列多 chiplet GPU 架构的基础。通过 chiplet 设计,AMD 可以在不增加单芯片面积的情况下扩展计算和内存资源。有源桥接设计是向更高效 chiplet 架构的演进。
来源: Google Patents - US20200409859A1, Google Patents - US20210097013A1

趋势展望

Chiplet 架构成为主流

NVIDIA (多 dielet MMU 同步)、AMD (Active Bridge)、华为 (四 chiplet 封装) 均在推进 chiplet GPU 设计。随着单芯片工艺逼近物理极限,chiplet 将成为提升计算密度和降低成本的必然选择。MMU 跨 chiplet 同步、一致性互连是关键技术挑战。

内存与计算解耦

NVIDIA FAMM、华为自研 HBM、AMD Helios 机架级内存池化,均指向同一趋势:内存容量和带宽不再受限于单 GPU。数据中心将独立扩展内存和计算资源,降低 TCO。缺陷芯片重用作内存控制器是 NVIDIA 的成本创新。

计算-通信重叠深化

从 NVIDIA Copy Engine Collectives、AMD SDMA 卸载,到华为依赖感知任务拆分和 FlashOverlap wave-group 同步,计算-通信重叠正从"粗粒度 kernel 级"向"细粒度 warp/tile 级"演进。CCCL 的压缩耦合进一步减少了通信量。

开放互连标准 vs 专有生态

AMD UALink 倡导开放标准,华为计划开放灵衢 2.0 规范,而 NVIDIA NVLink Fusion 在开放平台的同时保持控制。开放标准可能打破 NVIDIA 的生态锁定,但 NVLink 的性能和生态优势仍然显著。

光互连技术崛起

AMD 自 2017 年投资硅光子学,华为灵衢全光互联方案,均指向光 I/O 替代铜互连的趋势。随着 AI 集群规模从千卡向十万卡扩展,光互连在带宽密度和能效上的优势将愈发明显。

MoE 专用通信库

DeepSeek DeepEP 在 MoE All-to-All 通信中超越 NCCL,表明通用集合通信库在特定工作负载下的效率不足。随着 MoE 模型成为主流,专用通信库将成为重要发展方向。
