Bug NVLink 活跃 Issues

#971 — B200 nvidia-smi 挂起 + NVLink Rx Detect 错误

Updated Jul 6

问题: B200 GPU 在 driver 570.133.20 OpenRM 下运行约 66 天后，nvidia-smi 无限挂起。日志显示 NVLink 错误 "Failed to update Rx Detect Link mask!"。该 issue 已被 NVIDIA 标记为 NV-Triaged，持续活跃。

来源: Issue #971

环境信息

GPU: NVIDIA B200 | Driver: 570.133.20 OpenRM | Kernel: 6.6.0 | 创建日期: 2025-11-22 | 状态: Open (NV-Triaged)

分析

该问题指向 Blackwell 数据中心 GPU 的 NVLink 链路稳定性问题。Rx Detect 是 PCIe/NVLink 链路训练的关键步骤，失败表明物理层存在间歇性通信故障。长时间运行后触发说明可能与温度漂移、信号衰减或固件状态机超时有关。NVLink inband 消息机制可能需要增加 link retrain 容错逻辑。

#1208 — RTX 5090 (Blackwell) Xid 31 MMU FAULT_PDE 页表损坏

Jun 17

问题: 3x RTX 5090 上，约 20 次独立 CUDA 程序编译后触发 FAULT_PDE (Page Directory Entry 错误)，导致 GPU 虚拟地址空间损坏。日志: "MMU FAULT_PDE ... ACCESS_TYPE_VIRT_READ"。仅 Blackwell (sm_120) 受影响，Ampere/Volta 无此问题。

来源: Issue #1208

分析

GPU code cache 条目累积来自不同 JIT 编译过程，导致页表损坏。FAULT_PDE 表示 Page Directory Entry 无效，这是 MMU 页表遍历（page table walk）中的关键结构。Blackwell 特有的 GMMU (GPU MMU) 实现可能存在 code cache 回收路径的 bug。

环境: Driver 610.43.02, CUDA 13.3, Ubuntu 26.04, JAX/XLA

#1187 — BAR1 VA-Space 耗尽 → 系统硬锁 (Turing)

Jun 9

UVM UVM 关键 Issues

#1223 — nvidia_uvm HMM Fault-Service Eviction 死锁

Jul 1

问题: HMM 内存超额订阅场景下，GPU 满载 + replayable fault eviction 触发 nvidia_uvm kthread 永久自旋 (RCU stall)。进程不可杀，仅重启可恢复。环境: RTX PRO Blackwell, Driver 595.71.05 open, Kernel 6.14.0-1015-nvidia。复现: cudaMallocManaged > VRAM 并触摸所有页面。

来源: Issue #1223

分析

UVM (Unified Virtual Memory) 的 HMM (Heterogeneous Memory Management) 路径在处理 page fault 时存在死锁。当 GPU 内存完全被占用且存在大量 replayable faults 时，eviction 路径与 fault-service 路径形成循环依赖。RCU stall 表明内核调度器被阻塞，这是严重的系统级问题。

关键路径: kernel-open/nvidia-uvm/uvm_mmu.c → HMM fault handler → eviction → deadlock

Hot 高频 Issues 与社区讨论

eGPU 稳定性问题集中爆发

高频

本周 eGPU (Thunderbolt/OCuLink) 相关问题活跃：#1229 (PCIe timeout → MMU Fault) 和 #1228 (GSP RTD3 电源状态修复) 形成配套 issue/PR 对。社区反馈 RTX 5090 eGPU 场景下驱动缺乏 PCIe error handler 支持，导致瞬态错误升级为致命故障。

来源: #1229 | #1228

eGPU 社区对 NVIDIA 驱动的外接 GPU 支持长期不满。核心问题：

  • • 缺少 pci_error_handlers 注册，无法处理 PCIe AER 错误
  • • GSP 固件错误地在外接 GPU 上广告 RTD3 深度休眠状态
  • • Thunderbolt/OCuLink enclosure 无法进行 slot power-gating

PR #1228 通过 _kgspInitGpuProperties() 中检查 PDB_PROP_GPU_IS_EXTERNAL_GPU 来修复电源状态问题。

Blackwell MMU 页表损坏引发关注

高频

Issue #1208 揭示了 Blackwell (sm_120) 特有的 MMU FAULT_PDE 问题。多次 CUDA JIT 编译后 GPU 虚拟地址空间损坏，仅 Blackwell 受影响。社区担忧这是 GMMU 硬件实现或驱动页表管理的新 bug。

来源: #1208

该问题在 JAX/XLA 工作负载中尤为突出，因为 XLA 编译器会为每个模型变体生成独立的 CUDA 代码。约 20 次编译后触发，说明 GPU code cache 的 PDE (Page Directory Entry) 管理存在资源泄漏或回收缺陷。

VA-Space 耗尽模式跨代复现

趋势

Issue #1187 (Turing) 和 #1140 (Ada) 展示了相同的 dmaAllocMapping_GM107 VA-space 耗尽模式。从 Turing 到 Ada Lovelace 跨两代 GPU 复现，说明这是驱动层面的 VA-space 管理缺陷而非硬件限制。

来源: #1187 | #1140

两个 issue 的共同特征：长时间运行（7天+或 WebGL 持续使用）后 dmaAllocMapping_GM107 无法分配 VA space，随后 RC watchdog 检测到 GPU 锁死。这表明驱动的 VA-space 回收机制存在泄漏，长期运行场景下最终导致不可恢复故障。

Linux 7.x 内核兼容性挑战

兼容

Linux 7.0.x 和 7.2 的发布给 NVIDIA 驱动带来兼容性挑战：#1233 报告 7.0.x 的 "Unpatched return thunk" 警告；#1227 为 7.2 适配 strncpy 移除和 DRM atomic API 重命名。社区 PR 积极推动上游兼容。
