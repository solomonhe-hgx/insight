二、技术背景与要解决的问题

2.1 背景

现代 GPU 计算系统（如数据中心 AI 训练/推理集群）面临一个核心矛盾：计算能力与内存容量之间的耦合。传统架构中，每个 GPU 拥有自己的本地高带宽内存（HBM），内存容量受限于单个 GPU 封装的物理面积和成本。当应用需要远超单 GPU 内存容量的数据时，传统方案（如统一内存、页面迁移）存在延迟高、带宽低、效率差等问题。

2.2 要解决的问题

专利要解决的核心问题包括：

  1. 内存容量扩展瓶颈：如何在不增加计算 GPU 成本的前提下，大幅增加系统可用的 GPU 可访问内存总量。
  2. 原子操作的硬件支持：远程内存在执行 read-modify-write 等原子操作时，如何避免软件模拟带来的性能损失。
  3. 非对称连接的寻址挑战：源 GPU 通过大量链路（如 k=12 条 NVLink）连接到 NVSwitch 织物，而每个 FAM 模块仅通过少量链路（如 2 条）连接，导致地址空间远大于单个 FAM 模块的寻址能力。
  4. 成本效率：如何以低成本实现大容量远程内存池。

三、核心发明概述

本专利提出了一个完整的 织物附加存储（Fabric Attached Memory, FAM） 系统架构，其核心创新可归纳为以下四个层面：

3.1 Floor-Swept GPU 作为内存控制器

这是本专利最具商业洞察力的发明。NVIDIA 提出将制造过程中存在缺陷的 GPU 芯片（"floor-swept"，即"扫地"级别的残次品）进行改造：

  • 禁用计算引擎：通过熔断（fuse）或软件配置，故意禁用 GPU 的流式多处理器（SM）等计算电路。
  • 保留关键接口：保留内存控制器、NVLink 织物接口、数据缓存、交叉开关等必要组件。
  • 重新定位为内存控制器：这些残次 GPU 被封装到 FAM 模块（FAMM）中，充当低成本、高性能的内存控制器。

关键优势：这些 GPU 虽然计算能力有缺陷，但其内存控制器和 NVLink 接口通常是完好的。利用它们作为内存控制器，天然支持 GPU 原子操作（atomic operations）和对等通信（peer-to-peer），无需额外的定制芯片。

3.2 地址转换流水线（Swizzle → Compaction）

这是本专利最核心的技术创新，解决了非对称连接下的寻址问题。整个地址转换过程如下：

源 GPU 虚拟地址 (VA)
    ↓  [源 GPU MMU 翻译]
织物线性地址 (FLA / FAMLA) — 全局地址空间，如 0~12 GB
    ↓  [源 GPU Swizzle]
Swizzled 地址 — 使用伽罗瓦域数学对地址位进行变换，产生熵用于链路选择
    ↓  [NVLink 链路选择]
通过 k 条 NVLink 中的一条发送到 NVSwitch
    ↓  [NVSwitch 入口路由表查找]
    ↓  [De-swizzle：逆向 Swizzle 变换]
    ↓  [Compaction：地址压缩，除以链路数 L]
FAM 压缩地址 (FAMCA) — 压缩后的地址空间，如 0~2 GB
    ↓  [Donor GPU 最终翻译]
物理内存地址

3.2.1 Swizzle（源 GPU 端）

源 GPU 在发送内存请求前，对地址执行 swizzle 操作：

  • 使用伽罗瓦域（Galois Field）数学对地址的特定比特位进行混合变换。
  • 目的：使连续的地址请求分散到不同的 NVLink 链路上，实现负载均衡（专利中称为"喷洒/spraying"）。
  • 这确保了多条链路被充分利用，避免单链路过载。

3.2.2 De-swizzle + Compaction（NVSwitch 端）

NVSwitch 在收到请求后执行反向操作：

  • De-swizzle：逆向执行源 GPU 的 swizzle 变换，恢复原始地址。
  • Compaction（压缩）：将大的全局地址空间（FAMLA）除以连接到该 FAM 的链路数 L，消除 swizzle 产生的间隙，生成紧凑的 FAMCA 地址。例如，如果全局地址空间为 12 GB，FAM 通过 L=2 条链路连接，则压缩后每个 FAM 看到的地址空间为 0~2 GB（假设 6 个 FAM 共享 12 GB 空间）。

3.2.3 为什么需要这个机制

源 GPU 通过 k=12 条 NVLink 连接到 NVSwitch，而每个 FAMM 仅通过 L=2 条链路连接。源 GPU 看到的是全局的 12 GB 地址空间（FAMLA），但单个 FAMM 的物理内存只有 2 GB。如果不做压缩，FAMM 的内存控制器需要处理 12 GB 的地址范围，其中大部分是空洞（无效地址），这会浪费 TLB 和页表资源。Compaction 消除了这些空洞，使 FAMM 只需处理紧凑的 2 GB 地址空间。

3.3 NVSwitch 路由机制（Map Slot）

NVSwitch 使用 入口路由表（Ingress Port Route Remap Table） 实现地址路由，表中的每个条目称为一个 Map Slot。每个 Map Slot 包含以下关键字段：

字段 功能
Base（基址） 地址范围的起始值，用于匹配检查
Limit（界限） 地址范围的结束值，用于匹配检查
TgtID（目标标识） 指定请求应路由到哪个 FAMM 目标，软件可编程
Shuffle Mode 控制 de-swizzle 和 compaction 的具体模式
Offset 可编程偏移量，加到压缩后的地址上，用于映射到 FAMM 内的正确物理位置

路由流程：

  1. NVSwitch 入口端口收到内存请求。
  2. 从请求地址中提取特定比特位，与 Map Slot 表中的 Base/Limit 比较。
  3. 找到匹配的 Map Slot 后，读取其中的 TgtID，确定目标 FAMM。
  4. 根据 Shuffle Mode 字段执行 de-swizzle + compaction。
  5. 加上 Offset 字段值，得到最终的 FAMCA 地址。
  6. 将请求转发到 TgtID 指定的 FAMM。

这个机制完全由硬件实现，通过软件（驱动程序）预先配置路由表，运行时无需 CPU 介入。

3.4 数据条带化（Data Striping）

除了硬件层面的"喷洒"（spraying），专利还定义了软件层面的数据条带化策略：

  • 物理条带（Physical Stripe）：内存地址在多个 FAMM 之间按"条带"分配。例如，FAMM0 负责地址 0~2 GB，FAMM1 负责 2~4 GB，以此类推。
  • 水平条带 vs 垂直条带：
  • 水平条带（Horizontal）：连续地址块分配给不同 FAMM。
  • 垂直条带（Vertical）：地址按更细粒度的交错方式分配。
  • 逻辑条带（Logical Stripe）：物理条带地址空间可进一步划分为多个逻辑条带，用于支持多节点系统。每个节点拥有自己独立的逻辑条带，互不干扰。
  • 条带共享模型：专利还描述了不同节点间共享条带的模型，允许一个节点访问另一个节点的 FAM 资源。

4.2 源 GPU（Source GPU）

源 GPU 是完整的计算 GPU，具备：

  • 流式多处理器（SM）用于执行计算线程。
  • MMU（内存管理单元）负责将虚拟地址翻译为织物线性地址（FLA/FAMLA）。
  • Swizzle 引擎，对地址执行伽罗瓦域变换。
  • k 条 NVLink 发射链路（如 k=12），用于向 NVSwitch 发送请求。
  • 支持 peer aperture 扩展，使远程 FAM 内存映射到源 GPU 的物理地址空间中。

4.3 FAM 模块（FAMM）

每个 FAM 模块包含：

  • Donor GPU（捐赠 GPU / Floor-swept GPU）：核心组件，包含：
  • Boot ROM：启动固件。
  • DDR 内存控制器：硬件加速原子操作（atomic），无需软件模拟。
  • Row Remapper：行重映射器，用于内存坏行管理。
  • Data Cache：数据缓存。
  • Crossbar（交叉开关）：内部数据通路。
  • Fabric Interconnect Interface：NVLink 织物接口，支持对等通信。
  • 计算引擎被禁用（通过熔断或软件配置）。
  • 离散 DRAM 设备阵列：提供大容量内存（如 16 GB 或更多）。
  • L 条 NVLink 链路（如 L=2）：连接到 NVSwitch。

4.4 FAM 服务处理器

每个 FAM 基板上有一个专用的 FAM 服务处理器 CPU：

  • 管理基板上的所有 FAMM。
  • 运行与主机 CPU 不同的操作系统。
  • 负责内存管理、错误处理、固件更新等。
  • 通过独立的控制总线（非 NVLink）与 FAMM 通信。

4.5 非对称连接

本系统的关键特征是非对称连接：

  • 源 GPU → NVSwitch：k 条链路（如 k=12），提供高带宽。
  • FAMM → NVSwitch：L 条链路（如 L=2），较少链路。
  • 这意味着 k > L，源 GPU 的发射带宽远大于单个 FAMM 的接收带宽。
  • 这种非对称性正是需要 swizzle + compaction 机制的根本原因。

五、地址翻译全流程详解

以一个具体场景为例：源 GPU 0 要写入 FAMM 2 的某个地址。

阶段 1：虚拟地址 → 织物线性地址（FAMLA）

  • 源 GPU 的 MMU 将线程发出的虚拟地址（VA）翻译为织物线性地址（Fabric Linear Address, FLA），也称为 FAMLA。
  • FAMLA 是一个全局地址，覆盖所有 FAMM 的总内存空间。例如，6 个 FAMM 各 2 GB，则 FAMLA 范围为 0~12 GB。
  • 这个翻译通过扩展的 peer aperture 机制实现——FAM 的内存被映射到源 GPU 的物理地址空间中。

阶段 2：Swizzle（源 GPU 执行）

  • 源 GPU 的 swizzle 引擎对 FAMLA 地址执行伽罗瓦域变换。
  • 变换后的地址低位用于选择 k=12 条 NVLink 链路中的一条。
  • 目的：使请求均匀分布到所有链路上，避免热点。
  • 例如，连续地址 0, 1, 2, 3, ... 可能被分配到链路 3, 7, 1, 9, ...（伪随机分布）。

阶段 3：NVSwitch 入口路由

  • 请求通过选定的 NVLink 链路到达 NVSwitch 的某个入口端口。
  • NVSwitch 查找该入口端口对应的 Route Remap Table。
  • 从请求地址中提取特定比特位作为索引，找到对应的 Map Slot。
  • Map Slot 中的 Base/Limit 字段验证地址是否在有效范围内。
  • TgtID 字段确定目标 FAMM（本例中为 FAMM 2）。

阶段 4：De-swizzle + Compaction（NVSwitch 执行）

  • NVSwitch 根据 Map Slot 的 Shuffle Mode 字段，执行 de-swizzle（逆向 swizzle）恢复原始 FAMLA。
  • 然后执行 compaction：将 FAMLA 除以 L（链路数），消除地址间隙。
  • 例如：FAMLA = 8 GB（FAMM 2 的地址范围为 8~10 GB 的全局空间），compaction 后 FAMCA = 0~2 GB。
  • 加上 Map Slot 中的 Offset 字段值，得到最终的 FAMCA 地址。

阶段 5：Donor GPU 最终翻译

  • 压缩后的请求（带 FAMCA 地址）通过 L=2 条链路之一到达 FAMM 2。
  • Donor GPU 的内存控制器将 FAMCA 地址翻译为物理 DRAM 地址。
  • 执行请求的内存操作（读/写/原子操作）。
  • 结果沿原路返回源 GPU。

六、权利要求书分析

本专利共有 16 项权利要求，可分为以下五组：

第一组：设备/装置权利要求（Claims 1-7）

Claim 1（独立权利要求）：
一种织物附加存储（FAM），包含：
- 一个配置为与互连织物通信的处理器；
- 至少一个与该处理器操作性耦合的内存；
- 该处理器被构造为在所述内存上执行至少一个 read-modify-write 原子内存访问命令；
- 关键限定：该处理器的计算电路能力被编程或以其他方式故意禁用。

分析：这是最核心的独立权利要求，覆盖了"用计算能力被禁用的 GPU 作为 FAM 内存控制器"这一基本发明构思。"read-modify-write 原子操作"的硬件支持是与普通内存控制器的关键区别。

Claim 2（从属权利要求）：计算电路通过熔断（fuse）被故意禁用。

Claim 3：处理器包含至少一个流式多处理器（SM）。（暗示这是 GPU）

Claim 4：互连织物包含 NVIDIA NVLink。

Claim 5：内存包含离散半导体随机存取存储器设备阵列。

Claim 6（独立权利要求）：
与 Claim 1 类似，但增加了额外限定：
- 处理器的计算电路能力"有缺陷、被禁用或不存在"（三种替代方案）。
- 处理器包含多个织物互连端口，仅一个子集被配置为连接到互连织物。

分析：Claim 6 扩大了保护范围——不仅覆盖"故意禁用"的情况，还覆盖"天生有缺陷"或"根本没有"计算电路的情况。"仅子集端口连接"限定了非对称连接。

Claim 7（独立权利要求）：
进一步列举了处理器的具体组件：Boot ROM、内存控制器、行重映射器（row remapper）、数据缓存、交叉开关连接、织物连接。

分析：这是一个非常具体的独立权利要求，列出了 donor GPU 内部的实际组件清单，为专利侵权诉讼提供了多层次的保护。

第二组：系统权利要求（Claims 8-12）

Claim 8（独立权利要求）：
一种织物附加存储系统，包含：
- 互连织物；
- 至少一个连接到织物的源 GPU（生成内存地址）；
- 多个连接到织物的 FAM（各定义一个地址空间）；
- 关键限定：源 GPU 与织物的连接、以及每个 FAM 与织物的连接是非对称的；
- 至少包含一个地址变换器，将源 GPU 生成的内存地址变换为 FAM 地址空间。

分析：这是系统层面的核心独立权利要求，覆盖了完整的非对称连接架构和地址变换机制。

Claim 9：地址变换器包含除法或压缩电路。

Claim 10：地址变换器包含与源 GPU swizzle 匹配的解 swizzler 和地址压缩器。

Claim 11：源 GPU 执行 swizzle 以选择织物内的互连链路。

Claim 12：每个 FAM 的地址空间小于源 GPU 生成的地址空间。

第三组：基板权利要求（Claims 13-14）

Claim 13（独立权利要求）：
一种 FAM 基板，包含：
- 印刷电路板（PCB）；
- PCB 上的多个 FAM 模块，每个连接到互连织物；
- PCB 上的处理器，管理多个 FAM 模块；
- 关键限定：FAM 模块能执行 GPU 原子内存操作和对等 GPU 通信，同时将计算能力 GPU 的数量与内存容量解耦；每个 FAM 模块包含一个 floor-swept GPU（至少有缺陷和/或熔断以禁用 GPU 计算操作）。

分析：这是基板层面的独立权利要求，保护了完整的 FAM 基板产品设计。"将计算能力 GPU 的数量与内存容量解耦"是本专利的核心商业价值主张。

Claim 14（独立权利要求）：
与 Claim 13 类似的基板权利要求，但从另一个角度限定——FAM 模块包含"没有 GPU 计算能力"的内存控制器，并列举了具体组件：Boot ROM、硬件加速原子命令的内存控制器（无需模拟）、行重映射器、数据缓存、交叉互连、织物互连接口。

第四组：扩展系统权利要求（Claims 15-16）

Claim 15（独立权利要求）：
一种 FAM 系统，包含：
- 互连织物；
- 连接到织物的 GPU（提供计算能力）；
- 连接到 GPU 的第一内存（本地内存）；
- 处理电路（包含 Boot ROM、内存控制器、行重映射器、数据缓存、交叉开关、织物连接），能执行 read-modify-write 原子操作但被配置为不提供计算能力；
- 连接到处理电路的第二内存（远程 FAM 内存）；
- 关键限定：GPU 能够通过互连织物和处理电路原子访问第二内存。

七、关键技术创新点总结

7.1 商业模式创新：缺陷芯片的价值转化

将制造缺陷 GPU 转化为低成本内存控制器，是一个极具商业智慧的发明。它同时解决了两个问题：废弃芯片的处理成本和远程内存控制器的制造成本。Floor-swept GPU 天然支持 NVLink 协议和 GPU 原子操作语义，这是定制 ASIC 难以匹敌的竞争优势。

7.2 地址映射创新：Swizzle + Compaction 流水线

这个机制优雅地解决了非对称连接下的寻址难题。源 GPU 端的 swizzle 实现了链路负载均衡，NVSwitch 端的 de-swizzle + compaction 消除了地址间隙，使大地址空间透明地映射到小地址空间。整个过程对源 GPU 的软件栈完全透明。

7.3 路由创新：Map Slot 可编程路由

NVSwitch 的 Map Slot 机制提供了灵活的、软件可编程的硬件路由。通过配置 Base/Limit/TgtID/Shuffle Mode/Offset 字段，系统可以动态调整 FAM 拓扑，而无需修改硬件设计。

7.4 架构创新：计算与内存的解耦

通过 FAM 架构，NVIDIA 实现了计算资源（源 GPU）和内存资源（FAMM）的物理分离和逻辑统一。源 GPU 通过标准的内存操作（load/store/atomic）即可访问远程 FAM，就像访问本地 HBM 一样。

八、与相关技术的关系

8.1 与 CXL 内存扩展的区别

CXL（Compute Express Link）是业界标准的内存扩展方案，但本专利的方案有本质区别：
- CXL 基于 PCIe 协议栈，延迟相对较高。
- 本专利基于 NVLink/NVSwitch，延迟更低、带宽更高。
- 本专利的 FAM 天然支持 GPU 原子操作语义，CXL 内存通常不支持。
- 本专利使用 floor-swept GPU 作为控制器，成本可能低于 CXL 内存扩展器。

8.2 与 CUDA CFT 的关系

本专利（优先权日 2019-11-04）是 NVIDIA 后续推出的 CUDA Compute Fabric Transport（CFT）编程模型的硬件基础。CFT Handle 中的 leId（逻辑端点标识）和 offset（资源偏移）正是建立在本专利描述的 FAM 地址架构之上。CFT 的 fabric.try_get/fabric.try_put 等 PTX 指令，本质上是通过本专利的 swizzle + compaction 流水线来访问远程 FAM 内存。

8.3 被引用情况

根据专利数据库记录，本专利家族已被 37 项专利/申请引用，引用方包括 Samsung、Intel、Apple、AMD、HPE 等主要半导体公司，涉及内存池化、多 GPU 架构、神经网络训练加速等领域，表明本专利在 GPU 内存架构领域具有重要的基础地位。

九、专利保护策略分析

NVIDIA 在本专利中采用了多层次、多维度的保护策略：

  1. 设备层面（Claims 1-7）：保护 floor-swept GPU 作为 FAM 控制器这一设备本身。
  2. 系统层面（Claims 8-12）：保护包含源 GPU、NVSwitch 织物、FAM 的完整系统架构。
  3. 基板层面（Claims 13-14）：保护包含多个 FAMM 和管理处理器的完整基板产品。

  5. 分案策略：通过 US11822491B2 分案申请，追求与母案不同角度/范围的权利要求保护。

这种策略使得竞争对手很难通过简单的设计规避来绕过 NVIDIA 的专利保护。

十、总结

US11182309B2 / US11822491B2 是 NVIDIA 在 GPU 内存架构领域的一项基础性专利。它提出了一种完整的织物附加存储（FAM）系统，通过将制造缺陷 GPU 改造为低成本内存控制器、设计 swizzle + compaction 地址转换流水线解决非对称连接寻址问题、以及利用 NVSwitch 的可编程路由表实现灵活的内存拓扑管理。这项专利是 NVIDIA 在数据中心 GPU 内存解耦方向的重要技术布局，为后续 NVLink-based 内存池化和 CUDA CFT 编程模型奠定了硬件基础。
