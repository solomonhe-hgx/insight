US20230393788A1: 封装内/外组合内存系统 (Combined On-Package and Off-Package Memory System)

Memory System Memory MMU
申请公司
Nvidia Corp
申请日期
2023-08-23
公开日期
2023-12-07
专利号
US20230393788A1
核心创新: 提出一种组合内存架构，将封装内 HBM 内存与封装外 CXL/PCIe 连接的扩展内存统一管理。内存管理逻辑在逻辑基础层中实现地址翻译卸载，通过页表将请求引导至近端 (near) 或远端 (far) 内存池，实现分层内存池化，GPU 可通过单一地址空间访问不同层级的内存资源。
技术解读:

该专利的核心在于其内存管理逻辑 (memory management logic) 集成在封装的逻辑基础层中，能够自动完成地址翻译和请求引导。具体而言，逻辑基础层使用页表 (page table) 将处理器的统一地址空间映射到不同的物理内存池 -- 封装内 HBM 提供高带宽低延迟的热数据层，封装外 CXL/PCIe 内存提供大容量的冷数据层。

关键技术创新包括: (1) 交错地址映射 (interleaved address mapping) 将请求分散到多个内存设备以聚合带宽; (2) 迁移引擎 (migration engine) 在不同内存池之间透明迁移数据，不增加处理器接口流量; (3) 地址翻译卸载机制将页表遍历从处理器核心卸载到专用硬件。该专利直接支撑了 Grace Hopper Superchip 中的 Fabric Attached Memory 架构实现。

从系统架构角度看，该专利解决了 AI 加速器面临的 "内存墙" 问题 -- 封装内 HBM 容量有限 (目前最高 288GB) 但带宽高，封装外内存容量大但带宽低。通过硬件级的分层管理，GPU 软件栈无需修改即可透明使用两级内存。

影响分析: 该专利是 NVIDIA 内存解耦战略的核心实现。随着 LLM 模型参数量突破万亿级别，单 GPU 的 HBM 容量已无法满足需求。该专利使 GPU 不再受限于封装内 HBM 容量，为 Grace Hopper 和后续 Rubin 平台提供了内存扩展路径，对超大规模 LLM 训练和推理有深远影响。
来源: Google Patents - US20230393788A1

US11822491B2: Fabric Attached Memory 高效访问技术

FAM Memory NVLink
申请公司
Nvidia Corp
申请日期
2021-10-20
授权日期
2023-11-21
专利号
US11822491B2
核心创新: 提出 Fabric Attached Memory (FAM) 架构，将远程内存通过 NVLink/NVSwitch 互连 fabrics 附加到 GPU 地址空间。采用地址压缩 (address compaction) 和基于熵的链路选择 ("spraying") 技术，将请求分布到作为内存控制器的低规格 GPU 上，实现计算资源与内存资源的独立扩展。
技术解读:

FAM 模块使用 "低规格 GPU" (reduced-capability GPUs) -- 这些 GPU 不具备完整计算能力，但原生处理内存控制功能。系统通过 NVLink 互连使源 GPU 可以像访问本地内存一样使用原子函数访问 FAM 内存。地址翻译涉及 "swizzle" 操作以随机化链路选择，防止特定 NVLink 路径上的流量热点。

交换芯片执行 "compaction" 操作，将大的源 GPU 地址空间映射到较小的 FAM 模块地址范围，消除地址空洞。软件在多个 FAM 设备间分配 "stripes" 内存以最大化带宽并行性。该架构使内存容量可以独立于计算资源进行扩展，解决了传统 GPU 集群中计算与内存比例固定的问题。

从工程实现角度，FAM 的核心挑战在于保持远程内存访问的延迟和原子语义一致性。该专利通过硬件级的地址压缩和链路负载均衡，将远程内存访问的性能损失降到最低，同时保持了与本地内存相同的编程模型。

影响分析: FAM 是 NVIDIA 内存语义互连愿景的核心实现，使 GPU 集群从 "计算+通信" 模型演进为 "统一内存池" 模型。该专利已在 Grace Hopper Superchip 中首次量产，对分布式训练和 HPC 有深远影响。FAM 也为 NVIDIA 的 "GPU 即内存" 战略奠定了技术基础。
来源: Google Patents - US11822491B2

技术解读:

该专利对统一虚拟内存 (UVM) 场景尤为重要 -- 在 UVM 中，CPU 和 GPU 可能通过不同的虚拟地址访问同一物理内存区域。专利的控制机制确保在这些别名访问期间的缓存一致性，同时不牺牲性能。这对于 GPU 虚拟化 (vGPU) 和多进程共享内存场景也具有关键意义。

影响分析: 存储别名控制是 GPU 内存管理的基础性专利，直接影响 UVM、vGPU 和内存映射 I/O 的正确性和性能。随着多实例 GPU (MIG) 和 GPU 虚拟化在企业数据中心的普及，该专利的价值将进一步凸显。

US20230289189A1: 分布式共享内存 (Distributed Shared Memory / DSMEM)

DSMEM Comm Memory
申请公司
Nvidia Corp
申请日期
2022-03-10
公开日期
2023-09-14
专利号
US20230289189A1
核心创新: 提出协作组数组 (Cooperative Group Arrays, CGAs) 执行模型和分布式共享内存 (DSMEM) 机制。硬件保证 CGA 中所有 CTA 在多个 SM 上并发驻留，线程可直接访问同一 CGA 内其他 SM 的本地共享内存，实现跨 SM 的低延迟数据共享和同步，无需通过全局内存或 L2 缓存。
技术解读:

该专利引入了两个核心概念: (1) CGA 执行模型 -- 硬件级别的并发保证确保所有 CTA 在启动前共驻于机器上; (2) DSMEM -- 分布在多个 SM 上的共享内存块组成的统一内存池。DSMEM 支持跨 SM 的读、写和原子操作，使一个 SM 可以直接写入另一个 SM 的共享内存。

新的 ISA 指令支持 CGA 中所有运行 CTA 的所有线程的 barrier 同步。这使得 split-K reduction 等操作显著加快，因为一个 SM 可以直接写入另一个 SM 的共享内存。该架构为深度学习和 HPC 工作负载的 strong scaling 提供了硬件基础。

DSMEM 在 H100 (Hopper) 架构中首次实现，SM-to-SM 网络延迟远低于经过 L2 缓存的路径。该专利的编程模型通过 CUDA Cooperative Groups API 暴露给开发者，已在 CCCL (CUDA C++ Core Libraries) 中得到应用。

影响分析: DSMEM 是 NVIDIA 在多 GPU/多 SM 一致性内存方面的核心专利，为 H100/B100/GB200 等多芯片系统提供了硬件级同步原语。该专利直接支撑了 Tensor Parallelism 和 MoE 模型训练中的高效通信，进一步巩固了 NVIDIA 在 GPU 互连生态的技术壁垒。

US20250291730A1: 多芯片模块中的硬件辅助页迁移 (Hardware Assisted Page Migration in Multi-Dielet System)

Page Migration MMU
申请公司
Nvidia Corp
申请日期
2024-03-17
公开日期
2025-09-18
专利号
US20250291730A1
核心创新: 在多芯片模块 (MCM) GPU 中，每个 chiplet 上的硬件 MMU 机制感知所有 chiplet 上的引擎页表绑定 (engine page-table binds)。通过在 chiplet 间广播绑定信息，访问计数器可以准确关联引擎与绑定页面，实现精确的页迁移决策通知，避免跨 chiplet 的页表不一致问题。
技术解读:

随着 GPU 从单芯片向多芯片模块 (Multi-Chip Module / Multi-Dielet) 架构演进，内存管理面临新的挑战。每个 chiplet 拥有自己的 MMU 和页表，但当多个 chiplet 共享统一虚拟地址空间时，页迁移决策需要全局视角。该专利提出了一种硬件机制，在每个 chiplet 上维护对其他 chiplet 页表绑定状态的感知。

具体实现中，当一个 chiplet 上的 MMU 进行页表绑定时，绑定信息通过硬件广播机制传播到所有其他 chiplet。每个 chiplet 上的访问计数器 (access counter) 利用这些绑定信息来准确追踪哪些引擎在访问哪些页面，从而为页迁移策略提供精确的热度数据。这避免了传统方案中因跨 chiplet 页表信息不同步而导致的错误迁移决策。

该专利对 Blackwell 和 Rubin 等多芯片 GPU 架构至关重要。在这些架构中，不同 chiplet 可能拥有不同的内存本地性 (memory locality)，正确的页迁移策略直接影响性能。硬件级的绑定信息广播确保了迁移决策的准确性和及时性。

影响分析: 该专利直接面向 NVIDIA 的多芯片 GPU 未来路线。随着 Blackwell B200/B100 和 Rubin R100 采用多 chiplet 封装，硬件辅助页迁移成为确保 UVM 性能的关键。该专利与 ISCA 2025 上发表的 LATPC (局部性感知 TLB 预取) 研究互补，共同构成了 NVIDIA 下一代 MMU 架构的核心。
来源: Justia Patents - US20250291730, Google Patents - US20250291730A1

US20200356492A1: 虚拟化环境中的内存访问安全

Memory Security MMU NVLink
申请公司
Nvidia Corp
申请日期
2020-07-23
公开日期
2020-11-12
专利号
US20200356492A1
核心创新: 提出基于 Fabric Linear Address (FLA) 空间的多节点集群内存访问安全机制。每个处理节点独立虚拟化和管理本地内存访问，仅处理具有正确访问凭证的请求。通过多级地址翻译链 (VA -> FLA -> GPA -> SPA) 和 NVLink 集成，在保持高性能的同时实现进程间隔离。
技术解读:

该专利的核心在于 FLA (Fabric Linear Address) 空间设计 -- 一种跨节点的线性地址抽象。源节点将本地虚拟地址 (VA) 映射到 FLA，目标节点再将 FLA 映射到 Guest 物理地址 (GPA) 再到系统物理地址 (SPA)。这种多级翻译链确保了跨节点内存访问的安全隔离。

关键创新包括: (1) 权限在目标节点处授予或拒绝，确保隔离性; (2) 使用大 TLB 页面 (如 512 MB) 在源节点高效映射大的连续 FLA 范围到目标节点的碎片化物理页面; (3) NVLink 集成支持缓存一致性和直接内存访问，配合地址翻译服务 (ATS)。该方案在保持 NVLink 低延迟优势的同时，实现了类似 IOMMU 的安全隔离。

从安全角度看，该专利解决了 GPU 集群中一个关键问题: 当多个租户或进程共享 NVLink 互连的 GPU 集群时，如何防止跨进程的内存访问冲突和数据泄露。FLA 空间的设计使得每个进程只能访问被授权的内存区域。

影响分析: 该专利是 NVIDIA GPU 虚拟化和多租户安全的基础。随着 DGX 和 HGX 系统在云数据中心的广泛部署，多租户 GPU 隔离成为刚需。该专利的 FLA 机制为 NVIDIA vGPU 和 MIG 技术提供了硬件级安全支撑，确保不同 VM/容器间的内存访问不会相互干扰。
来源: Google Patents - US20200356492A1

US20230393788A1 (MMU 视角): 组合内存系统中的地址翻译卸载

MMU Offload MMU Memory
核心创新 (MMU 角度): 在组合内存系统中，逻辑基础层的 MMU 硬件卸载处理器的地址翻译功能。页表存储在基础层中，内存管理逻辑使用页表自动将请求引导到近端或远端内存池，实现硬件级的地址翻译和内存池路由。
技术解读:

从 MMU 设计角度看，该专利的核心创新在于将地址翻译逻辑从处理器核心移到封装内的专用硬件层。这类似于 CPU 世界中 IOMMU 的角色，但针对 GPU 的多池内存架构进行了优化。MMU 硬件维护多组页表，分别对应近端 (HBM) 和远端 (CXL/PCIe) 内存池，并根据地址范围自动选择翻译路径。

该设计对 GPU MMU 架构有重要影响: 传统 GPU MMU 只需处理单一物理内存的地址翻译，而该专利的 MMU 需要同时管理多个内存池的翻译，并在内存池之间进行透明的数据迁移。这要求 MMU 具备更复杂的页表结构和更高效的 TLB 管理策略。

影响分析: 该专利代表了 GPU MMU 从简单地址翻译向复杂内存管理引擎演进的趋势。未来 Rubin 平台的 MMU 可能需要支持更多层级的内存池 (HBM4 + CXL3 + NVLink 远程内存)，该专利的架构为此提供了基础框架。
来源: Google Patents - US20230393788A1

US20230289190A1: 可编程控制的多计算引擎数据多播

Data Multicast Comm
申请公司
Nvidia Corp
申请日期
2022-03-10
公开日期
2023-09-14
专利号
US20230289190A1
核心创新: 提出一种编程化多播技术，使一个线程可以代表其他多个线程请求数据。跟踪电路 (tracking circuitry) 在处理器核心和多播请求之间进行接口管理，优化 L2 缓存和 SM 带宽使用。该机制避免了多个线程重复请求相同数据造成的带宽浪费。
技术解读:

在 GPU 计算中，多个线程/线程块经常需要访问相同的数据 (如模型权重、共享参数)。传统方式下，每个线程独立发起内存请求，导致 L2 缓存和内存控制器的带宽浪费。该专利通过硬件多播机制，将相同数据的请求合并为一次内存访问，然后将结果分发给所有请求者。

多播由跟踪电路实现，该电路位于处理器核心和内存子系统之间。它接收来自多个核心的多播请求，追踪哪些线程等待同一数据，当数据到达时同时通知所有等待者。该机制对 MoE (Mixture of Experts) 模型特别重要，因为不同 token 可能路由到同一 expert，需要读取相同的 expert 权重。

从集合通信角度看，该多播机制可以视为 GPU 内部的 "硬件 AllGather" -- 一次内存读取服务多个消费者。这与 NCCL 的集合通信原语形成互补: NCCL 处理跨 GPU 通信，而该专利处理 GPU 内部跨 SM/线程的数据分发。

影响分析: 数据多播是 AI 推理和训练中的关键优化。随着 MoE 模型成为主流架构 (如 Mixtral, GPT-4 等)，多播技术可以显著减少 expert 权重的重复加载，提升 L2 缓存命中率和内存带宽利用率。该专利对 NVIDIA 在 AI 推理性能优化方面具有重要战略价值。
来源: Google Patents - US20230289190A1

US20230315655A1: 快速数据同步 (Speed-of-Light 同步)

Fast Sync Comm Sync
申请公司
Nvidia Corp
申请日期
2022-03-10
公开日期
2023-10-05
专利号
US20230315655A1
核心创新: 实现 "光速" (Speed-of-Light, SOL) 数据同步机制，将 GPU 间同步延迟降低至约 0.5 个往返时间 (RTT)。通过组合的 store-and-arrive 指令和分离的 producer/consumer barrier，消除了传统同步协议的软件开销，达到接近物理极限的同步速度。
技术解读:

传统 GPU 间同步 (如 MPI barrier、CUDA device synchronize) 需要完整的请求-确认往返 (1 RTT)，加上软件栈开销后延迟更高。该专利提出的 SOL 同步机制通过硬件-软件协同设计，将同步信息嵌入数据包头部，接收端解析即可获知同步状态，无需额外的确认往返。

核心机制包括: (1) 组合的 store-and-arrive 指令 -- 一次操作同时完成数据存储和到达通知; (2) 分离的 producer/consumer barrier -- 生产者和消费者可以独立管理各自的同步状态; (3) 硬件级的同步状态追踪 -- 无需软件轮询。这些优化使同步延迟接近物理极限 (单程传播延迟)。

该同步机制在 NVLink 和 InfiniBand 上均可实现，对 NCCL 集合通信原语有直接影响。AllReduce、AllGather 等操作中的 barrier 和 reduce 步骤都可以从 SOL 同步中受益，特别是在大规模集群中，同步延迟的降低会被通信轮数放大。

影响分析: 0.5 RTT 同步是分布式训练中的关键优化。在万卡集群 (如 100K+ GPU) 中，同步延迟往往是扩展瓶颈。该专利使 NVIDIA GPU 在通信原语层面达到接近物理极限的同步速度，巩固了 NVIDIA 在大规模分布式训练中的技术领先。该专利与 Meta 开源的 NCCLX 库中的优化方向一致。
来源: Google Patents - US20230315655A1

核心创新: 引入 "事务 barrier" (transaction barrier) 同步原语，由专用硬件单元管理跨处理器的异步事务并发。系统同时追踪线程到达和异步硬件事务 (如 DMA, TMA) 的完成状态。同步单元缓存 barrier 状态以减少内存带宽消耗，使用 "try-wait buffer" 在不使用软件轮询循环的情况下通知被阻塞的线程。
技术解读:

该专利解决了一个关键问题: 在异步数据搬运 (如 TMA - Tensor Memory Accelerator) 和计算重叠的现代 GPU 编程模式中，传统的 barrier 同步无法感知异步事务的完成状态。事务 barrier 将线程同步和事务完成追踪统一到一个硬件机制中。

核心设计包括: (1) 事务 barrier 寄存器 -- 同时追踪线程到达计数和未完成事务计数; (2) try-wait buffer -- 缓存 barrier 状态，当所有线程到达且所有事务完成时，自动唤醒等待的线程; (3) 跨分区支持 -- 可在不同 GPU 分区 (MIG 实例) 之间使用。这使 producer-consumer 模式可以高效地在异步事务环境中工作。

从编程模型角度看，该专利为 CUDA 的异步编程模型提供了硬件基础。开发者可以使用 cuda::barrier 和 TMA 指令组合，让硬件自动管理计算和数据搬运之间的同步，无需手动插入 fence 或等待指令。

影响分析: 事务 barrier 是 NVIDIA Hopper 和 Blackwell 架构中异步编程模型的关键硬件支撑。随着 TMA 在 GPU 编程中的广泛使用 (TensorRT-LLM, CUTLASS 等)，该专利的同步机制直接影响 AI 训练和推理的性能。该专利与 DSMEM 专利 (US20230289189A1) 形成互补，共同构成 NVIDIA 的异步同步框架。
来源: Google Patents - US20230289242A1

趋势展望

趋势一: MMU 从翻译引擎向管理引擎演进

从本周专利可以看出，NVIDIA 的 GPU MMU 正在从简单的虚拟-物理地址翻译引擎，演进为复杂的内存管理引擎。US20250291730A1 (硬件辅助页迁移) 和 US20230393788A1 (地址翻译卸载) 表明 MMU 正在承担越来越多的职责:

  • 跨 chiplet 页表一致性管理
  • 多内存池的自动路由和迁移
  • 安全隔离 (FLA 空间)
  • 访问计数和热度追踪

这一趋势将在 Rubin 平台中进一步加速，MMU 可能需要支持 HBM4 + CXL3.1 + NVLink 远程内存的三级内存池管理。

趋势二: 同步机制从软件向硬件迁移

NVIDIA 的同步专利族 (DSMEM, SOL, Transaction Barrier) 展示了一个清晰的趋势: 同步操作正在从软件实现向硬件加速迁移。这对编程模型有深远影响:

  • 软件 barrier 被硬件 barrier 替代，延迟降低数倍
  • 异步事务同步使计算和通信重叠更加高效
  • DSMEM 使跨 SM/GPU 的数据共享无需经过全局内存

这一趋势意味着 CUDA 编程模型将继续向异步化方向演进，cuda::barrier 和 TMA 将成为主流编程范式。

趋势三: 内存解耦与池化成为核心战略

FAM (US11822491B2)、组合内存系统 (US20230393788A1)、地址转换 API (US12499065B1) 和内存信息 API (US12474981B2) 共同构成了 NVIDIA 的内存解耦与池化战略:

  • GPU 不再受限于封装内 HBM 容量
  • 计算和内存可以独立扩展
  • 软件通过统一 API 访问异构内存资源
  • Vera Rubin 平台目标: 机架级内存 20.7TB

内存解耦将使 GPU 集群的 TCO (Total Cost of Ownership) 显著降低，因为可以根据实际需求独立配置计算和内存资源。

趋势四: 专利布局聚焦多芯片架构

本周收录的专利中有超过一半直接面向多芯片 (multi-chiplet / multi-dielet) 架构:

  • US20250291730A1 -- 多芯片模块页迁移
  • US20230393788A1 -- 封装内外组合内存
  • US11822491B2 -- Fabric Attached Memory
  • US12536056B2 -- 跨 GPU DSMEM 同步

这表明 NVIDIA 的专利布局已经全面转向多芯片架构。随着单芯片面积接近光刻极限 (reticle limit)，多芯片封装将成为 GPU 扩展的主要路径。NVIDIA 通过密集的专利布局，在 chiplet 互连、内存管理和同步机制方面建立了强大的知识产权壁垒。
